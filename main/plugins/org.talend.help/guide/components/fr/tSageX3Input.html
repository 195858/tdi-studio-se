tSageX3Input

Propriétés du tSageX3Input

Famille de composant

Business/Sage X3

 

Fonction

Le composant tSageX3Input tire profit du Service Web fourni par un serveur Web Sage X3 pour extraire des données du système Sage X3 (serveur X3).

Objectif

Le composant tSageX3Input extrait des données d’un système Sage X3 donné.

Basic settings

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier où sont stockées les propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Edit Schema

Un schéma est une description de lignes, il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (Built-in) , soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma. Notez que si vous effectuez des modifications, le schéma devient automatiquement built-in.

 

Endpoint address

Saisissez l’adresse du Service Web fourni par le serveur Web Sage X3 donné.

 

Username et Password

Saisissez les informations d’authentification pour le Service Web, définies lors de la configuration du Serveur Web Sage X3.

 

Language

Saisissez le nom du code langue X3 utilisé pour le démarrage du groupe de connexions.

 

Pool alias

Saisissez le nom du pool de connexion distribuant les requêtes reçues aux connexions disponibles. Ce nom est donné depuis la console de configuration Sage X3.

 

Request config

Saisissez la chaîne de caractères de configuration si vous souhaitez récupérer les informations de débogage ou de traces. Par exemple, la chaîne de caractères peut être :

RequestConfigDebug=“adxwss.trace.on=on”;

Si vous devez utiliser plusieurs chaînes de caractères, séparez-les avec un &, par exemple,

RequestConfigDebug=“adxwss .trace.on=on&adxwss.trace.size=16384”;

Avertissement

Un outil tiers est nécessaire pour récupérer ce type d’informations.

 

Publication name

Saisissez dans ce champ le nom de l’objet, de la liste ou du sous-programme publié(e) auquel/à laquelle vous souhaitez que votre Studio accède.

 

Mapping

Remplissez ce tableau pour mapper les éléments variables de l’objet, le sous-programme ou la liste défini(e)(s) dans le serveur Web Sage X3 donné. Les colonnes à compléter incluent :

Column : les colonnes définies dans l’éditeur de schéma de ce composant.

Group ID : l’identifiant de chaque groupe d’éléments variables. Par exemple, un groupe d’éléments variables peut représenter l’un des attributs d’un objet.

Field name : le nom de chaque élément variable.

 

Query condition

Cochez cette case pour paramétrer la (les) condition(s) de la requête. Les colonnes à compléter incluent :

Key : le nom des éléments variables utilisés comme clé de l’extraction de données.

Value : la valeur du champ clé donné utilisé pour extraire les données correspondantes.

 

Limit

Saisissez un chiffre ou un nombre indiquant le maximum de lignes de données à extraire.

Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du Job ainsi qu’au niveau de chaque composant.

Utilisation

Généralement utilisé comme composant de début. Un composant de sortie est nécessaire.

Scénario : Utiliser les clés des requêtes pour extraire des données d’un système Sage X3 donné

Ce scénario décrit un Job à deux composants utilisé pour extraire une ligne ou des données d’un système Sage X3 donné. La méthode des objets est appelée, les éléments variables de cet objet sont des attributs. Les données utilisées dans ce scénario proviennent de l’exemple fourni par Sage X3.

Configurer le Job

  1. Déposez un composant tSageX3Input et un tLogRow de la Palette dans l’espace de modélisation graphique.

  2. Reliez le composant tSageX3Input au tLogRow à l’aide d’un lien Row > Main.

Configurer le schéma du tSageX3Input

  1. Double-cliquez sur le tSageX3Input afin de configurer ses Basic settings dans la vue Component.

  2. Dans cet éditeur, cliquez douze fois sur le bouton [+] pour ajouter 12 lignes à la table.

  3. Saisissez les noms que vous souhaitez utiliser pour chaque ligne. Dans cet exemple, ces lignes sont nommées d’après les noms de publication des attributs des objets définis dans le serveur Web Sage X3. Ces colonnes sont utilisées pour mapper les champs des attributs correspondants dans le système Sage X3.

  4. Dans la colonne Type, cliquez sur la ligne IMG pour afficher une liste déroulante.

  5. Dans la liste déroulante, sélectionnez List, puisque l’attribut apparaît plusieurs fois dans Sage. Répétez l’opération pour changer le type des lignes TIT2NBLIG, ITMLNK et ZITMLNK en List pour la même raison.

  6. Cliquez sur OK pour valider les modifications et acceptez la propagation proposée par la boîte de dialogue qui s’ouvre.

Configurer la connexion au serveur Web Sage X3

  1. Dans le champ Endpoint address, saisissez l’URL du service Web fournie par le serveur Web Sage X3. Dans cet exemple, l’URL est : http://10.42.20.168:28880/adxwsvc/services/CAdxWebServiceXmlCC.

  2. Dans le champ User, saisissez le nom de l’utilisateur de Sage X3. Dans cet exemple, l’utilisateur est ERP.

  3. Dans le champ Language, saisissez le nom du code langue X3 utilisé pour le démarrage d’un groupe de connexions, FRA dans cet exemple.

  4. Dans le champ Pool alias, saisissez le nom du pool de connexion à utiliser. Dans cet exemple, le pool de connexion est appelé TALEND.

  5. Dans le champ Publication name, saisissez le nom de publication de l’objet à appeler. Dans ce scénario, le nom de publication est ITMDET.

Paramétrer le mapping et configurer la condition de la requête

  1. Dans les colonnes Group ID et Field name de la table Mapping, saisissez les valeurs correspondant à l’ID du groupe d’attributs et au nom de publication définis dans le serveur Web Sage X3. Dans cet exemple, les valeurs sont présentées comme suit :

    Note

    Dans la table Mapping, la colonne Column a été automatiquement renseignée avec les colonnes que vous avez créées dans l’éditeur du schéma.

  2. Cochez la case Query condition pour activer le tableau Conditions.

  3. Sous le tableau Conditions, cliquez sur le bouton [+] pour ajouter une ligne.

  4. Dans la colonne Key, saisissez le nom de publication associé à l’attribut de l’objet dont vous souhaitez extraire les données.

  5. Dans la colonne Value, saisissez la valeur de l’attribut que vous avez sélectionné comme clé de l’extraction. Dans ce scénario, sa valeur est CONTS00059, une des références produit.

Exécuter le Job

  1. Sélectionnez Built-In dans la liste Schema puis cliquez sur [...] à côté du champ Edit schema afin d’ouvrir l’éditeur du schéma.

  2. Appuyez sur F6 pour exécuter le Job.
