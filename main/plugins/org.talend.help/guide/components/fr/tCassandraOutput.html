tCassandraOutput

Avertissement

Ce composant est disponible dans la Palette du Studio si vous avez souscrit à l'édition correspondante de Talend Big Data Studio.

Propriétés du tCassandraOutput

Famille de composant

Big Data / Cassandra

 

Fonction

Le composant tCassandraOutput reçoit des données du composant précédent et les écrit dans Cassandra.

Objectif

Le tCassandraOutput est utilisé pour écrire des données dans Cassandra.

Basic settings

Use existing connection

Cochez cette case et sélectionnez le composant de connexion adéquat à partir de la liste Component list pour réutiliser les paramètres d’une connexion que vous avez déjà définie.

 

Host

Saisissez le nom de l'hôte ou l'adresse IP du serveur Cassandra.

 

Port

Saisissez le numéro du port d'écoute du serveur Cassandra.

Keyspace configuration

Keyspace

Saisissez le nom de l'espace de clé (Keyspace) dans lequel vous souhaitez écrire les données.

 

Action on keyspace

Sélectionnez l'opération que vous souhaitez effectuer sur l'espace de clé :

  • None : Aucune opération n'est effectuée.

  • Drop and create keyspace : Le keyspace est supprimé et créé à nouveau.

  • Create keyspace : Le keyspace n'existe pas et est créé.

  • Create keyspace if not exists : Le keyspace est créé s'il n'existe pas.

  • Drop keyspace if exists and create : Le keyspace est supprimé s'il existe déjà et est recréé.

 

Column family configuration

Saisissez le nom de la famille de colonnes dans laquelle vous souhaitez écrire les données.

 

Column family type

Standard : La famille de colonnes est de type standard.

Super : La famille de colonnes est de type super.

 

Action on column family

Sélectionnez l'opération que vous souhaitez effectuer sur la famille de colonnes :

  • None : Aucune opération n'est effectuée.

  • Drop and create column family : La famille de colonnes est supprimée et recréée.

  • Create column family : La famille de colonnes n'existe pas et est créée.

  • Create column family if not exists : La famille de colonnes est créée si elle n'existe pas.

  • Drop column family if exists and create : La famille de colonnes est supprimée si elle existe déjà et recréée.

 

Action on data

Sélectionnez l'opération à effectuer sur les données de la table définie :

  • Upsert : Insérer les colonnes si elles n'existent pas ou mettre à jour les colonnes existantes.

  • Delete : Supprimer les colonnes correspondantes du flux d'entrée.

 

Schema et Edit schema

Un schéma est une description de lignes, il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit Built-in, soit stocké dans le Repository.

Si vous utilisez Talend Open Studio for Big Data, seul le mode Built-in est disponible.

 

Sync columns

Cliquez sur ce bouton afin de récupérer le schéma du composant précédent dans le Job.

 

Row key column

Sélectionnez la colonne de clé de ligne dans la liste.

 

Include row key in columns

Cochez cette case pour inclure la clé de ligne dans les colonnes.

 

Super columns

Sélectionnez la super colonne, dans la liste.

Cette liste déroulante apparaît si vous sélectionnez Super dans la liste Column family type.

 

Include super columns in standard columns

Cochez cette case pour inclure les super colonnes dans les colonnes standards.

 Delete row

Cochez cette case pour supprimer la ligne.

Cette case apparaît uniquement lorsque vous sélectionnez Delete dans la liste déroulante Action on data.

 Delete columns

Sélectionnez ou personnalisez les colonnes à supprimer.

 Delete super columns

Cochez cette case pour supprimer des super colonnes.

Cette case apparaît uniquement si vous cochez la case Delete Row.

Advanced settings

Batch Size

Nombre de lignes dans chaque lot traité.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du Job ainsi qu'au niveau de chaque composant.
