tWebService

Avertissement

Ce composant requiert l’utilisation d’un JDK Oracle.

Propriétés du tWebService

Famille de composant

Internet

 

Fonction

Ce composant appelle la méthode du service Web invoqué et retourne la classe définie en fonction des paramètres donnés.

Objectif

Le tWebService invoque une méthode via un service Web afin de récupérer les valeurs des paramètres sélectionnés dans l’éditeur.

Basic settings

Property type

Peut être Built-in ou Repository.

Si vous utilisez Talend Open Studio for Big Data, seul le mode Built-in est disponible.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier dans lequel sont stockées les propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Service configuration

Cliquez sur le bouton [...] pour ouvrir l’éditeur du composant. Dans l’éditeur, vous pouvez :

-sélectionner le service Web que vous souhaitez utiliser,

-configurer les paramètres d’entrée du service Web,

-configurer les paramètres de sortie du service Web permettant de récupérer les données à transmettre en sortie.

 

Mapping links display as

Auto : par défaut, les liens faisant la correspondance entre les schémas d’entrée et de sortie et les paramètres du service Web sont en forme de courbes.

Curves : les liens faisant la correspondance entre les schémas d’entrée et de sortie et les paramètres du service Web sont en forme de courbes.

Lines : les liens faisant la correspondance entre les schémas d’entrée et de sortie et les paramètres du service Web sont en forme de lignes droites. Cette dernière option améliore légèrement les performances.

 

Connection Time out

Saisissez en secondes la durée de connexion au service Web.

 

Receive Time out

Saisissez en secondes le temps de réponse du serveur.

 

Schema et Edit schema

Un schéma est une description de lignes, il définit le nombre de champ qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma. Notez que si vous modifiez le schéma, il devient automatiquement built-in.

Cliquez sur Sync columns pour récupérer le schéma à partir du composant précédent dans le Job.

Si vous utilisez Talend Open Studio for Big Data, seul le mode Built-in est disponible.

 

 

Built-in : Le schéma est créé et conservé pour ce composant seulement.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé.

 Sync columnsCe bouton est disponible lorsqu'un lien d'entrée est créé. Cliquez sur ce bouton pour récupérer le schéma du composant précédent.
 Output schema

Ce champ est utilisé pour traiter le schéma de sortie. Le schéma est soit built-in, soit distant dans le Repository. Il est configuré comme le schéma d'entrée.

Avertissement

Le schéma d'entrée n'est pas nécessairement identique à celui du schéma de sortie.

Si vous utilisez Talend Open Studio for Big Data, seul le mode Built-in est disponible.

 

Use NTLM

Cochez cette case si vous utilisez un protocole d’authentification NTLM.

Domain : Nom de domaine du client,

Host : Adresse IP du client.

 

Need authentication

Cochez la case d’authentification et renseignez le nom de l’utilisateur dans le champ Username et son mot de passe dans le champ Password, si cela est nécessaire pour accéder au service.

 

Use http proxy

Cochez cette case si vous vous connectez derrière un proxy et renseignez les informations correspondantes.

 

Trust server with SSL

Cochez cette case pour authentifier le serveur auprès du client via un protocole SSL et renseignez les champs correspondants.

TrustStore file : saisissez le chemin d’accès et le nom du fichier TrustStore contenant la liste des certificats approuvés par le client.

TrustStore password : saisissez le mot de passe utilisé pour vérifier l’intégrité des données TrustStore.

 

Die on error

Décochez cette case pour passer les lignes en erreur et terminer le traitement des lignes sans erreur.

Advanced settings

Temporary folder (for wsdl2java)

Définissez (ou parcourez votre répertoire jusqu’à) un dossier temporaire configuré où stocker les fichiers wsdl.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

 

Enable parallel execution

Cocher cette case permet de traiter plusieurs flux de données simultanément. Cela permet de traiter les données plus rapidement.

Pour plus d'informations, consultez le Guide utilisateur de Talend Open Studio.

Utilisation

Ce composant peut être utilisé comme composant intermédiaire ou comme composant de début de Job. Il requiert un composant de sortie.

Limitation

Un JDK est requis pour faire fonctionner ce composant.

Scénario : Extraire une liste de noms à l’aide d’un service Web

Ce scénario décrit un Job à trois composants utilisant un service Web pour récupérer la liste des défenseurs de l’équipe de football d’un pays donné.

Vous pouvez également appeler un Job exporté en tant que service Web. Pour plus d'informations, consultez la section intitulée « Scénario 2 : Mettre les données de sortie en mémoire tampon du serveur d’application Web » et la section intitulée « Scénario 4 : Appeler un Job exporté en tant que service Web dans un autre Job ».

  • A partir de la Palette, cliquez-glissez les composants suivants dans l’espace de modélisation : tFixedFlowInput, tWebService et tLogRow.

  • Reliez les composants à l’aide de liens de type Main Row.

  • Double-cliquez sur le composant tFixedFlowInput pour paramétrer ses propriétés dans la vue Component.

  • Cliquez sur le bouton [...] à côté du champ Edit schema pour définir le schéma du composant d’entrée.

  • Dans la boîte de dialogue Schema of tFixedFlowInput, cliquez sur le bouton [+] pour ajouter une colonne au schéma.

  • Renommez cette colonne Country.

  • Cliquez sur OK pour valider. Une boîte de dialogue s’ouvre et vous demande si vous souhaitez propager les modifications.

  • Cliquez sur Yes (Oui).

  • Dans le tableau Values, saisissez la valeur d’entrée. Dans ce scénario, saisissez la valeur “ITALY” pour récupérer la liste des défenseurs de l’équipe d’Italie.

  • Double-cliquez sur le composant tWebService pour ouvrir l’interface dédiée ou cliquez sur le bouton [...] de l’onglet Basic settings de la vue Component.

Dans l’onglet WSDL, connectez-vous au service Web à utiliser dans le Job.

  • Dans le champ WSDL, saisissez l’adresse WSDL du service Web que vous souhaitez utiliser ou cliquez sur le bouton Browse... pour sélectionner un WSDL stocké localement.

  • Cliquez sur le bouton Refresh pour récupérer la description du WSDL dans les champs suivants.

  • Dans la liste Port Name, sélectionnez le port que vous souhaitez utiliser. Ici, sélectionnez le port FootballPoolWebServiceSoap.

  • Dans la liste Operation, sélectionnez le service que vous souhaitez utiliser. Ici, sélectionnez le service AllDefenders(parameters):ArrayOfString.

  • Cliquez sur Next pour passer à l’étape suivante.

Dans l’onglet Input mapping, faites la correspondance entre le schéma du composant de début et le service Web.

Dans la zone de droite, le paramètre d’entrée du service apparaît automatiquement. Vous avez la possibilité d’ajouter d’autres paramètres en sélectionnant l’élément [-] parameters de la zone de droite, en cliquant sur le bouton [+], et en sélectionnant les paramètres souhaités dans la boîte de dialogue [Parameter Tree].

Le service Web utilisé dans ce scénario ne comporte qu’un seul paramètre d’entrée : sCountryName.

Note

S’il est disponible, utilisez le bouton Auto map! situé en bas à gauche de la fenêtre, il permet d’effectuer l’opération de mapping automatiquement.

Vous devez créer une connexion entre le schéma d’entrée et les paramètres d’entrée du Service Web défini. Pour ce faire :

  • Dans la liste Column, sélectionnez la colonne du schéma d’entrée que vous souhaitez mettre en correspondance avec le paramètre de sortie du service Web et glissez-la dans le paramètre correspondant dans la zone de droite.

  • Cliquez sur Next pour ouvrir une nouvelle vue dans l’éditeur.

Dans l’onglet Output mapping, faites la correspondance entre le paramètre de sortie du service Web et le schéma du composant de sortie.

Dans la liste Element à gauche de l’éditeur, le paramètre de sortie du service Web apparaît automatiquement. Vous avez la possibilité d’ajouter d’autres paramètres en sélectionnant l’élément [+] parameters de la liste Element à gauche de l’éditeur, en cliquant sur le bouton [+], et en sélectionnant les paramètres souhaités dans la boîte de dialogue [Parameter Tree].

Le service Web utilisé dans ce scénario ne comporte qu’un seul paramètre de sortie : AllDefendersResult.string.

  • Dans la zone de droite, cliquez sur le bouton [...] du champ Edit Schema pour définir le schéma de sortie.

  • Dans la zone Output à droite de la boîte de dialogue, cliquez sur le bouton [+] pour ajouter une colonne au schéma de sortie.

  • Nommez cette colonne Name. Elle récupèrera les noms des défenseurs.

  • Cliquez sur OK.

  • Dans l’éditeur du tWebService, sélectionnez l’élément parameters.AllDefendersResult.string à gauche et glissez-le dans le champ correspondant à la colonne Name dans la zone de droite.

Note

S’il est disponible, utilisez le bouton Auto map! situé en bas à gauche de la fenêtre, il permet d’effectuer l’opération de mapping automatiquement.

  • Sélectionnez cette ligne dans la zone de droite et cliquez sur le bouton Denormalize pour dénormaliser les données retournées.

  • Ajoutez le code [*] après le paramètre pour obtenir le code suivant : denormalize(parameters.AllDefendersResult.string[*],”:”) permettant de récupérer la totalité des données séparée par deux points “:”.

  • Cliquez sur OK.

  • Dans l’espace de modélisation, double-cliquez sur le composant tLogRow pour paramétrer ses propriétés dans la vue Component.

  • Cliquez sur le bouton Sync columns pour récupérer le schéma du composant précédent.

  • Enregistrez votre Job et appuyez sur F6 pour l’exécuter.

Les noms des défenseurs italiens sont retournés et affichés dans la console de Talend Open Studio.
