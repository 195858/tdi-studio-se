Scenario: Unduplicating entries

Based on the tSortRow job, the tUniqRow component is added to the job in order to uniquify the entries in the output flow. In fact, as the input data is randomly created, duplication cannot be avoided.

    • On the Properties tab panel of the tUniqRow component, click Edit Schema... to set the Key on Names field to uniquify the output flow on this criteria.
    • Check the Case Sensitive box to differenciate lower case and upper case.
    • Press F6 to run the job again. The console displays the sorted and unique results
