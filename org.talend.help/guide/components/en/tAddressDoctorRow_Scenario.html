An AddressDoctor component

The AddressDoctor component used with Talend Open Studio or Talend Integration Suite is tAddressDoctorRow. It detects spelling and formatting errors and do all necessary corrections.

Below is an example of a three-component job created in one of Talend studios. This job reads an address from an input file, uses the tAddressDoctorRow component to correct the address and apply some formatting on it, and finally writes the correct formatted address in the relevant database.

To be able to realize such example, you must be familiar with designing jobs and using Talend components. For more information, check Talend Open Studio User Guide and Talend Open StudioComponents Reference Guide respectively.

After defining the properties of the tFileInputDelimited and tMysqlOutput components, double-click tAddressDoctorRow to open its Basic settings view and define its properties.

To get a User Id and Password, you need first to sign up for an AddressDoctor Data Quality Center account at https://www.addressdoctor.com/AcctMngr/newaccount.aspx?lang=en. Then you will receive your User Id and Password by email. With this register-for-free operation, you can process limited number of addresses. If you need to process a higher number of addresses, you can purchase transactions online at http://www.addressdoctor.com.

The Origin Country option controls the output of the formatted address. Whenever mail or other items are sent cross border, it is necessary to place the country of destination in the address.

The below table describes the parameters for this option.

Parameter

Description

COO_ALWAYS_USE_DESTINATION_COUNTRY (by default)

The destination country will always be used in the formatted address

COO_NEVER_USE_DESTINATION_COUNTRY

The country of destination will never be used in the formatted address

COO_ABW

The country of origin is Aruba
...
 

COO_XXX

The country of origin is XXX where XXX is equal to the ISO3 country code

The Country Type option controls the way the country name is returned in the formatted address.

The below table describes the parameters for this option.

Parameter

Description

ISO_2

Returns the country as the ISO2 country code
ISO_3
Returns the country as the ISO3 country code

ISO_NUMBER

Returns the country as the numeric ISO code

ABBREVIATION

Returns the country abbreviation as often used in car license plates

NAME_CN

Returns the name in Chinese

NAME_DA

Returns the name in Danish
...
 

The line separation character must be set when passing or retrieving data that contain multiple lines.

The below table describes the parameters for the Line Separator option.

Parameter

Description

LF

Uses a linefeed as the line separation character.
You can also use this value when you would prefer a carriage return linefeed.The carriage return linefeed is not supported for technical reasons
SEMICOLON
Uses a semicolon as the line separation character

COMMA

Uses a comma as the line separation character

TAB

Uses a TAB character as the line separation character

NO_SEPARATOR

Uses no special line separation character. Fields will only be separated by a space

Press F6 to execute the job.

As you see in the above capture, the old address has two typos, one in the street name (paggess>pages) and the second in the city name (Suresne>Suresnes).

The tAddressDoctorRow component does not only correct your address, it also applies address normalization (as in the row ADDRESSDOCTOR_FORMATTED_ADDRESS) and address formatting (splitting the old address only row to many rows each representing a unit in the address, for example, STREET, POSTAL_CODE, LOCALITY and so on).
