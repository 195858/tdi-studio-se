tSQLiteRollback

tSQLiteRollback properties

This component is closely related to tSQLiteCommit and tSQLiteConnection. It usually does not make much sense to use these components independently in a transaction.

Component family

Databases/SQLite

 

Function

tSQLiteRollback cancels the transaction committed in the connected DB.

Purpose

Avoids to commit part of a transaction involuntarily.

Basic settings

Component list

Select the tSQLiteConnection component in the list if more than one connection are planned for the current Job.

 

Close Connection

Clear this check box to continue to use the selected connection once the component has performed its task.

Advanced settings

tStatCatcher Statistics

Select this check box to collect log data at the component level.

Usage

This component is to be used along with SQLite components, especially with tSQLiteConnection and tSQLiteCommit.

Limitation

n/a

Related scenarios

For tSQLiteRollback related scenario, see the section called “Scenario: Rollback from inserting data in mother/daughter tables”.
