tMergeFields

This component is available in the Palette of the studio on the condition that you have subscribed to the relevant edition of Talend Data Quality.

tMergeFields Properties

Component family
Data Quality
Function
tMergeFields receives a flow and merges it based on one or more columns. The aggregation key and the relevant result of operations (such as min, max, sum etc) are provided for each output line.
Purpose
tMergeFields centralizes data from various and heterogeneous sources. It helps to create a master copy of data for MDM.
Basic settings
Schema and Edit Schema
A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository.
 
 
Built-in: The schema will be created and stored locally for this component only. Related topic: Setting a built-in schema of Talend Integration Suite Studio User Guide.
 
 
Repository: The schema already exists and is stored in the Repository, hence can be reused in various projects and job designs. Related topic: Setting a Repository schema of Talend Integration Suite Studio User Guide.
 
Key
Define the merge sets, the values of which will be used for calculations.
Output column: Select the column name from the list that reflects the schema structure you defined. You can add as many output columns as you wish to make more precise aggregations.
Input column: Match each input column name with your output columns, in case the output column of the aggregation set needs to be different.
The columns in the Key table must NOT appear in the Operations table. If you want all the columns of the output schema to be filled in, they must appear either in the Key table or in the Operations table.
 
Operations
Output column: Select from the list the output column that will result from the selected merge operation.
Function: Select from the list the type of the merge operation to perform. The list includes count, min, max, avg, sum, first, last, list, list(objects), count(distinct), standard deviation, max length and replace.
Input column: Select from the list the input column from which the values are selected for the merge operation.
Weight column: Available only with the replace function. Select from the list the column you want to use as a weight value for the merge operation. Then the input column will be replaced with the value having the higher weight.
Ignore null values: Select the check boxes corresponding to the names of the columns for which you want the NULL value to be ignored.
Advanced settings
Delimiter (only for list operation)
Enter, between double quotation marks, the delimiter you want to use for the list operation.
 
Use financial precision, this is the max precision for "sum" and "avg" operations, checked option heaps more memory and slower than unchecked.
This check bow that enables to use financial precision is selected by default. Clear the check box if you want to use less memory and thus optemize performance.
 
Check type overflow (slower)
Checks the data type to ensure that the job does not crash.
If you select this check box, the system will be slower.
 
Check ULP (Unit in the Last Place), ensure that a value will be incremented or decremented correctly, only for float and double types. (slower)
Select this check box to launch the ULP verification.
If you slelect this check box, the system will be slower.
 
tStatCatcher Statistics
Select this check box to collect log data at the Job and the component levels.
Usage
This component is not startable and requires an input component and an output component.
