tLaunchDQReports

This component will be available in the Palette of the studio on the condition that you have subscribed to the relevant edition of Talend Data Quality.

tLaunchDQReports properties

Component family
Data Quality
Function
tLaunchDQReports launches existing reports in Talend Data Quality studio without having to open the Talend Data Quality platform. It does the same action as the Generate report file menu item in the Talend Data Quality platform.
Purpose
tLaunchDQReports can communicate with Talend Data Quality in order to launch an analysis and save the results of the analysis in a database. Using this component, you can launch the generation of all types of reports: pdf, XML, or HTML.
For information about how to generate a report file, see Generating a report file in Talend Data Quality User Guide.
Basic settings
Talend Data Quality executable file
Type in or copy-paste the path to the studio executable file (the .exe file used to launch your Studio).
 
Project Directory
Type in or copy-paste the path to the project which the report of interests belongs to.
 
Report filenames
Type in or copy-paste the relative technical file path of the report to be generated. You can set several report filenames separated by commas.

You can display the relative technical file path of the report you want to generate in the Detail view area below DQ Repository tree view in the Data Profiler perspective of your Studio when you select the relevant report in the tree view.

Advanced settings
tStatCatcher Statistics
Select this check box to collect log data at the component level.
Usage
This component is used as a stand-alone component to launch reports.

You are recommended to generate a launching-report Job using this component automatically from the DQ Repository tree view in the Data Profiler perspective of your Studio. For further information about how to generate this Job automatically, see section Generating a Job to launch a report in your Talend Data Quality User Guide.

In order to be executable, a Job containing this component must have an access to Talend Data Quality studio executable file. If the Job is to be used in a jobserver, the latter needs to access Talend Data Quality studio executable file (e.g.TalendDataQuality-win32-x86.exe on Windows).
