tSqoopExport

This component will be available in the Palette of the studio on the condition that you have subscribed to the relevant edition of Talend Integration Suite MPx.

tSqoopExport Properties

Component family
File/Sqoop
Function
tSqoopExport calls sqoop to transfer data from the Hadoop Distributed File System (HDFS) to a relational database management system (RDBMS).
Purpose
tSqoopExport is used to define the arguments required by Sqoop for transferring data to a RDBMS.
Basic settings
Connection
Type in the JDBC URL used to connect to the target database.
 
Table Name
Type in the name of the target table which data is tranfered to from HDFS. This table must already exist in the target database. The input files are read and parsed into a set of records according to the user-specified delimiters.
 
Export Dir
Type in the path, or browse to the source data to be transferred in HDFS.
 
User name and Password
Type in the authentication information to the target database.
 
Specify Number of Mappers
Select this check box to precise the number of mappers, which is the degree of parallelism used.
 
Print Log
Select this check box to activate the Verbose check box.
 
Verbose
Select this check box to print more information while working, for example, the debugging information.
Advanced settings
Direct
Select this check box to use the export fast path.
 
Use MySQL default delimiters
Select this check box to use MySQL's default delimiter set
 
Additional arguments
Complete this table to use an additional argument.
At each time, only one additional argument is allowed. Use of multiple additional arguments at the same time may lead to exception error..
 
tStatCatcher Statistics
Select this check box to collect log data at the component level.
Usage
This component is used standalone.

Incoming links (from one component to another):
