tGoogleMapLookup

Warning

This component will be available in the Palette of the studio on the condition that you have subscribed to the relevant edition of Talend Data Quality.

tGoogleMapLookup properties

Component family

Data Quality/Address

 

Function

tGoogleMapLookup accesses the Google Maps API over HTTP request to obtain detailed geographic information according to the address and geographic coordinates you provided.

Note

For further information about Google Maps Javascript API, see The Google Maps Javascript API V3.

Purpose

tGoogleMapLoopup allows you to obtain detailed gepgraphic information using geographic coordinates and address information.

Basic settings

Pause(in 1/10 seconds)

Time break between http requests for geographic information record. By default, the value for this field is 1.

 

Schema and Edit Schema

A schema is a row description, i.e., it defines the number of fields to be processed and passed on to the next component. The schema is either built-in or stored remotely in the Repository.

Usage

This component is used as an in-between component. It requires an input component, an output component and row Main link.

Scenario: Obtaining detailed geographic information using address and geographic coordinates

This scenario describes a three-component Job which send HTTP request to Google Geocoding API and obtain detailed geographic information based on the address and geographic coordinates we provided.

  1. Drop the following components from the Palette onto the design workspace: tFixedFlowInput, tGoogleMapLookup and tLogRow

  2. Connect tFixedFlowInput to tGoogleMapLookup and tGoogleMapLookup to tLogRow using Main links.

  3. Double-click tFixedFlowInput to display its Basic settings view to define its properties.

  4. Click Edit schema to open a dialog box that displays the data structure of input data and define the data you want to pass to tGoogleMapLookup, four columns in this scenario, lat, lng, address_id, and address.

  5. Set the Type of column lat and column lng to Double. Leave rest of the settings as they are.

  6. Click OK to close the dialog box.

  7. Click Use Single Table in the Mode area of the Basic settings view of tFixedFlowInput.

  8. In the Mode area of the Basic settings view of tFixedFlowInput, click the plus button to add four columns to the table.

  9. Fill in the field in the lat column with 37.5300125 and the lng column with "-122.2624254". And fill in the field in the address_id column with "1" and the address column with "500 Oracle Parkway, Redwood Shores, CA 94065".

  10. Double-click tGoogleMapLookup to display its Basic settings view and define its properties.

  11. Click Sync columns to retrieve the schema set in tFixedFlowInput and let the rest of settings as they are.

  12. Double-click tLogRow to display its Basic settings view and define its properties.

  13. Click Sync columns to retrieve the schema set in tGoogleMapLookup.

  14. Select Vertical (each row is a key/value list) in the Mode area and Print unique name in the Title printing mode area.

  15. Save your Job and press F6 to execute it.

    Note

    A limitation of 2500 geolocation requests from the same IP address per day is applied by Google though no Google API key is required by Geocoding API (V3). For further information about the usage limits of Geocoding API, see Usage Limits part in The Google Geocoding API.

The detailed geographic information of address 500 Oracle Parkway,Redwood Shores, CA 94065 is obtained from the Google Map database and printed onto the console.
