tQASAddressUnknown properties

Component family
Data Quality
Function
tQASAddressUnknown collects unknown addresses. It iterates on each row and reads all input addresses against the QuickAddress data.
Purpose
tQASAddressUnknown can give one output flow: Unknown that collects all addresses that do not match to deliverable results in the QuickAddress data.
Basic settings
QAS WSDL url
Enter the URL for the QuickAdress XML document. The URL is provided by Experian QAS.
 
Country
Select from the list the country corresponding to your input addresses.
 
Schema and Edit Schema
A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository.
 
 
Built-in: You create the schema and store it locally for this component only. Related topic: How to set a built-in schema of Talend Integration Suite Studio User Guide.
 
 
Repository: You have already created the schema and stored it in the Repository. You can reuse it in various projects and job designs. Related topic: How to set a repository schema of Talend Integration Suite Studio User Guide
 
Column to analyze
Select from the list the address column you want to analyze.
