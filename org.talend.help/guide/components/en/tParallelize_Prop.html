tParallelize

tParallelize displays as a component on the design workspace. However, its use is slightly different from typical components.

The tParallelize component does not process data itself nor flows of data, but is meant to help you parallelize and synchronize the execution of numerous subjobs in your main Job.

tParallelize Properties

Component family
Orchestration
Function
tParallelize allows to synchronize the execution of one subjob with the simultaneous execution of other subjobs in your master Job.
Purpose
tParallelize helps mastering complex job systems. It executes two or more subjobs simultaneously and then synchronizes the execution of a subjob, in the same master Job, with the simultaneous execution(s).
Basic settings
Wait For
end of first subjob: sequence the relevant subjob to be executed at the end of the first subjob.
 
 
end of all subjobs: sequence the relevant subjob to be executed at the end of all subjobs.
 
Sleep Duration
Set the time interval in seconds between each check for a subjob execution.
Usage
This component can be used as either a start or middle component in a master Job built of numerous subjobs. It can be connected to previous/next components using OnSubjobOk, Parallelize or Synchronize links. You can use as many tParallelize components as you want in your master Job.
