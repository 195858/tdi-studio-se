Scenario: Remotely launching a report from a given Talend Administration Center Web application

This scenario shows the collaborative operations performed to execute remotely an established launching-report Job from a given Talend Administration Center Web application. To do this, you need to commit all of the required metadata to the remote SVN repository used by this given Talend Administration Center Web application and need to install an execution server on the system hosting your Studio.

Prerequisites

Before starting this scenario, proceed as follows to set up the prerequisites:

      • Create a project from your Talend Administration Center Web application. The collaborative operations for launching a report will be performed in this project. In this scenario, this project is called test_ychen.
        For further information about how to create a project on the Talend Administration Center Web application, see section Adding a project in the Talend Administration Center User Guide.



        • If you have not the required rights to create or manage a project, see the administrator of your Talend Administration Center Web application.

      • Establish a connection from your Studio to the remote SVN repository storing the above-mentioned project.
        For further information about how to create the connection of this kind, see section How to connect to a remote repository in the Talend Integration Suite User Guide.
      • Launch your Studio using this new connection and the new project created. For further information about how to launch the Studio in this way, see section How to connect to a remote repository in the Talend Integration Suite User Guide.
      • In the Data Profiler perspective of your Studio, create a report which will be launched and generated remotely in this scenario. In this scenario, it is US_State.

For further information about how to create a report in the Data Profiler perspective, see section Creating a new report in the Talend Data Quality User Guide.

      • In the report editor in the Data Profiler perspective of your Studio, add the analysis, US_State in this scenario, upon which you base the report of interest on and type in the parameters in order to generate this report in the PDF format and store it in the following directory:
        C:\Documents and Settings\ychen\Bureau\TEST_YCHEN\Report_Center.

For further information about how to set up the relative parameters for report generation, see section Creating a new report in the Talend Data Quality User Guide.

If you did not create a report database to keep the history of the reports yet, your Studio will prompt you to create one. For further information, see section Managing the report database in the Talend Data Quality User Guide.

This way, the required metadata such as the analysis, the report and the DB connection have been committed to the remote SVN repository used by your Talend Administration Center Web application.

      • On the host system of your Studio, install and launch a jobserver. For further information, see chapter Installing services in your Studio's Installation Guide.

Launching a report remotely

After having prepared these prerequisite conditions, proceed as follows to replicate this scenario:

      • From the DQ Repository tree view of the Data Profiler perspective, right-click the US_State report under the Report node and select Generate Jobs > Launch a report from the contextual menu. This generates a launching-report Job.



        • It is recommended to verify the Basic settings of the tLaunchDQReport component used in this Job to ensure that the generated paths point to your Talend Data Quality executable file, to the report of interests and to the project this report belongs to.

      • In the menu tree view of your Talend Administration Center Web application, expand Job Conductor and click Servers to display the list of the remote execution servers available for deploying and executing Jobs.
      • From the toolbar on the Servers page, click Add to clear the Execution server configuration panel.
      • In this configuration panel, type in and save the configuration information of the jobserver you have installed and launched on the system hosting your Studio.
        For further information about the required configuration to connect to a jobserver, see Adding an execution server in the Talend Administration Center User Guide.

In this scenario, this jobserver's label is Report_Server, its host is 10.42.20.246 and the relative ports are 8000, 8001, 8888.

      • Click Save to validate this configuration. The new server displays in the server list of the Servers page and the green ticks indicate that the connection to this server is done.
      • In the menu tree view of your Talend Administration Center Web application, expand Job Conductor and click Execution task to display the task list on the Job Conductor page.
      • From the toolbar on the Job Conductor page, click Add to clear the Execution task configuration panel.
      • On this configuration panel, type in the parameters required for executing the launching-report Job you have generated.
      • In the Label field of this Execution task panel, type in the task name that will be displayed in the execution task list.
      • In the Project field, select the project which the launching-report Job to be used was generated in. In this scenario, it is the test_ychen project.
      • In the Branch field, select the branch of this project. It is trunk in this scenario.
      • In the Job field, select the launching-report Job to be used.
      • In the Version and the Context fields, select the information corresponding to this launching-report Job. In this scenario, they are 0.1 and Default.
      • In the Execution server field, select the jobserver which executes this task you are configuring. This server must be installed on the system hosting your Studio. In this scenario, this server is Report_Server you have added to the server list.
      • Click Save to validate this configuration. The new task displays in the task list on the Job Conductor page.
      • From the task list, select the newly added task.
      • On the toolbar of the Job Conductor page, click Generate.
      • Once the status of this task reads Ready to send on the task list, select this task again and click Deploy on the toolbar of the Job Conductor page.
      • Once the status of this task reads Ready to run on the task list, select this task again and click Run on the toolbar of the Job Conductor page.

Once the Job execution is done, check the output folder that you have defined in the Generated Report Settings view of the report editor in the Data Profiler perspective of your Studio.

In this scenario, the path to this output folder is
C:\Documents and Settings\ychen\Bureau\TEST_YCHEN\Report_Center
