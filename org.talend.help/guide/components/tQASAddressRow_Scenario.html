QuickAddress components

The QuickAddress components used in Talend Open Studio or Talend Integration Suite include: tQASAddressIncomplete, tQASAddressRow, tQASAddressVerified, and tQASAddressUnknown.

The tQASAddressIncomplete is an intermediate component that filters addresses. It can give two output flows: Incomplete and Reject. Incomplete collects all addresses that match to a partially-complete address. Reject collects all addresses that match to several possible results or do not match to deliverable results in the QuickAddress data.

The tQASAddressRow is an intermediate component that verifies columns in an address. It gives one output flow: Main. It iterates on each row and reads the input addresses against the QuickAddress data and:

      • corrects any formatting or spelling errors,
      • gives the verification status for each row since the address may not always have enough information to be matched to a single deliverable result in the QuickAddress data.

For more information about the verification status, see QuickAccess verification levels (verification status).

The tQASAddressVerified is an intermediate component that verifies addresses. It can give three output flows: Verified, Interaction required, and Reject. Verified collects all addresses matched to a single deliverable address in the QuickAddress data. Interaction required collects all addresses that matched to a single deliverable address in the QuickAddress data, however user interaction is still recommended to confirm that it is the correct address. Reject collects all addresses that do not match to deliverable results in the QuickAddress data.

The tQASAddressunknown is an intermediate component that fetches unknown addresses. It gives one output flow: Unknown. It collects all addresses that do not match to deliverable results in the QuickAddress data.

Below is an example of a five-component job created in one of Talend studios.

This job:

      • reads an input csv file that holds some client-related information,
      • uses the tMap component to concatenate the three fields Address, Postal, and City from the incoming data flow in one output column: Edit_Address,
      • uses the tQASAddressRow component to analyze the output column Edit_Address and gives the verification status of all edited addresses,
      • uses a tFilterRow components to output only the addresses which status is not equal to None.
      • and finally displays the correct formatted address along with their verification status on the console.



        • To be able to realize such example, you must be familiar with designing jobs and using Talend components. For more information, check Talend Open Studio User Guide and Talend Open StudioComponents Reference Guide respectively.

      • Click and drop the following components from the Palette onto the workspace: tFileInputDelimited, tMap, tAddressRow, tFilterRow, and tLogRow.
      • Double-click the tFileInputDelimited to open its Basic settings view and define the component properties.

The input file in this example is a cvs file that holds some French client personal information.

      • Double-click the tMap component to open the Map Editor and concatenate the Address, Postal, and City fields from the incoming data flow in one output column: Edit_Address.
      • Click Ok to validate and close the Map Editor.
      • Double-click the tQASAddressDoctor component to display its Basic settings and define its properties.
      • In the QAS WSDL url field, enter the URL for the QuickAdress XML document (provided by Experian QAS).
      • On the Country list, select the country corresponding to your input addresses (France in this example).
      • Click Sync columns to retrieve the schema set in the tMap component.
      • If necessary, click Edit schema to view the input and output data flow. The output schema should include the Edi_Address column that hold the Address, Postal, and City initial input columns.



        • The output schema of any of the QuickAddress components depends on the selected country since every country has different address norms.

      • Click OK to close the dialog box.
      • On the Column to analyze list, select Edit_Address.
      • Double-click the tFilterRow component to display its Basic settings view and define its properties.
      • In the Conditions area, click the plus button to add one condition to the output flow and in the corresponding fields:
        -select the input column you want to operate on,
        -select the needed function on the list,
        -select the operator to bind the input column with the value,
        -type in between the quotes the address value to be filtered.

In this example, we want to exclude the addresses which status is equal to None.

      • Double-click the tLogRow component to display its Basic settings and define its properties.

In this example, and for clarity purposes, we want the result to display on the console in a separate key/value tabular list for each row.

      • Save your job an press F6 to execute the job and display the result on the console.

In the above result samples, the tQASAddressRow reads the input rows, corrects and formats the addresses, gives the result in the Edit_Address row, and gives the verification status in the Status row.
