tLaunchDQReports

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l'édition correspondante de Talend Data Quality.

Propriétés du tLaunchDQReports

Famille de composant
Data Quality
Fonction
Le tLaunchDQReports lance des rapports existants dans le studio de Talend Data Quality sans avoir besoin d'ouvrir la plateforme Talend Data Quality. Il fait la même action que l'objet Generate report file du menu de la plateforme Talend Data Quality.
Objectif
Le tLaunchDQReports peut communiquer avec Talend Data Quality pour lancer une analyse et sauvegarder les résultats de l'analyse dans une base de données. En utilisant ce composant, vous pouvez lancer la génération de tous types de rapports : PDF, XML, ou HTML.
Pour plus d'informations sur la génération d'un fichier de rapport, consultez Générer un rapport existant dans le Guide Utilisateur de Talend Data Quality.
Basic settings
Talend Data Quality executable file
Chemin d'accès au fichier exécutable du studio.
 
Project Directory
Saisissez ou copiez-collez le chemin d'accès au projet auquel appartient le rapport qui vous intéresse.
 
Report filenames
Copiez le chemin d'accès technique du rapport à générer depuis le studio de Talend Data Quality et copiez-le dans ce champ. Vous pouvez paramétrer différents chemins d'accès en les séparant par une virgule.

Vous pouvez afficher le chemin d'accès technique du rapport que vous souhaitez générer, dans la zone Detail view sous l'arborescence DQ Repository dans le studio Talend Data Quality, quand vous sélectionnez le rapport correspondant dans l'arborescence.

Advanced settings
tStatCatcher Statistics
Cochez cette case pour collecter les métadonnées de process du Job, aussi bien au niveau du Job qu'au niveau de chaque composant.
Utilisation
Ce composant est utilisé en standalone. Il peut lancer des rapports depuis le studio de Talend Data Quality.

Il est recommandé de générer automatiquement un Job de lancement de rapport utilisant ce composant à partir de la vue DQ Repository dans la perspective Data Integration de votre Studio. Pour plus d'informations concernant la génération automatique d'un Job, consultez Générer un Job pour lancer un rapport, page 295 du Guide utilisateur de Talend Data Quality.

Pour qu'il soit exécutable, un Job contenant ce composant doit avoir un accès au fichier exécutable du studio Talend Data Quality. Si le Job est utilisé en Jobserver, il nécessitera un accès au fichier exécutable du studio Talend Data Quality (par exemple, TalendDataQuality-win32-x86.exe sous Windows).
