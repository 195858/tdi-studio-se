tSetDynamicSchema

Propriétés du tSetDynamicSchema

Famille de composant

Custom Code

 

Fonction

Le composant tSetDynamicSchema vous permet de configurer un schéma dynamique pouvant être réutilisé par des composants du (des) sous-job(s) suivant(s) afin de récupérer des données de colonnes inconnues.

Objectif

Le tSetDynamicSchema est utilisé pour créer un schéma dynamique permettant aux composants incapables de retrouver la structure des données gérées à partir des séparateurs de champs, comme le tFileInputPositional et le tFileOutputPositional, de récupérer les données provenant de colonnes inconnues.

Basic settings

Schema et Edit schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (Built-in) soit distant dans le Repository.

Cliquez sur Edit schema pour modifier le schéma. Notez que si vous modifiez le schéma, il devient automatiquement built-in.

  Built-in : Le schéma sera créé et conservé pour ce composant seulement. Voir également le Guide utilisateur de Talend Enterprise Studio.
  Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et Jobs. Voir également le Guide utilisateur de Talend Enterprise Studio.
 Sync columnsPermet de synchroniser le schéma du fichier d’entrée avec le schéma du fichier de sortie. La fonction Sync ne s’affiche que lorsqu’une connexion de type Row est connectée au composant de sortie.
 Parameters

Ce tableau contient deux colonnes.

Property : Propriété de l'entrée du schéma : ColumnName, DBName, Type, DBType, Length, Precision, Nullable ou Key.

Value : Valeur assignée à cette propriété.

Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du Job ainsi qu'au niveau de chaque composant.

Utilisation

Ce composant est généralement utilisé comme composant de fin dans un sous-job.

Limitation

Ce composant nécessite un composant d'entrée afin de définir la structure des données à configurer dans le schéma dynamique.

Scénario associé

Pour un scénario associé utilisant le composant tSetDynamicSchema, consultez la section intitulée « Scénario 2 : Gérer un fichier positionnel à partir d'un schéma dynamique » du composant tFileInputPositional.
