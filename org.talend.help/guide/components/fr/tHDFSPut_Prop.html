tHDFSPut

Propriétés du tHDFSPut

Famille de composant
File/Hadoop
Fonction
tHDFSPut charge les données dans le système de fichiers Hadoop.
Objectif
tHDFSPut se connecte au système de fichiers distribués Hadoop pour charger de gros fichiers avec une performance optimisée.
Basic settings
Host
Adresse IP du serveur du système de fichiers distribués Hadoop.
 
Port
Numéro du port d'écoute du seveur HDFS.
 
User name
Identifiant de l'utilisateur HDFS
 
Group
Identifiant de l'utilisateur et nom du groupe sous lesquels les instances HDFS ont été lancées.
 
HDFS directory
Emplacement des fichiers source dans HDFS.
 
Local directory
Dossier local où sont stockés les fichiers obtenus de HDFS.
 
Overwrite file
Option permettant d'écraser ou non les fichiers avec le nouveau fichier.
 
Files
Dans la zone Files, les champs à remplir sont :
- File mask : saisissez le nom du fichier qui sera sélectionné dans HDFS. Vous pouvez utiliser des expressions régulières.
- New name: renommez le fichier obtenu.
 
Die on error
La case est cochée par défaut. Décochez la case afin d'ignorer la ligne d'erreur et finir le processus de lignes sans erreur.
 
tStatCatcher Statistics
Cochez cette case pour collecter les données de log au niveau du composant.
 
Enable parallel execution
Cochez cette case permet de traiter plusieurs flux de données simultanément. Cela permet de traiter les données plus rapidement.
Pour plus d'informations, consultez Paralléliser les flux de données du Guide Utilisateur Talend Open Studio.
Les actions sur la table (Action on table) ne sont pas disponibles avec la fonction de parallélisation. Ainsi, pour créer une table, vous devez utiliser un composant tCreateTable.
Usage
Le composant combine la connexion HDFS et l'extraction de données, de cette façon il est souvent utilisé comme simple composant sous-job mais peut également être utilisé comme sortie ou comme composant final d'un Job.
Limitations
tHDFSGet et tHDFSPut peuvent être exécutés sous JRE 1.6+.
