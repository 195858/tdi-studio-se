tExtractEBCDICFields

Propriétés du tExtractEBCDICFields

Famille de composant

Processing/Fields

 

Fonction

Le composant tExtractEBCDICFields génère de multiples colonnes à partir d'une colonne, à l'aide de la mise en correspondance de regex.

Objectif

Le tExtractEBCDICFields vous permet d'utiliser des expressions régulière pour extraire des données d'une chaîne de caractères formatée.

Basic settings

Field

Sélectionnez dans la liste Field un champ entrant à extraire.

 

Die on error

Cette case est cochée par défaut et stoppe le Job en cas d’erreur. Décochez cette case pour terminer le traitement avec les lignes sans erreur, et ignorer les lignes en erreur. Vous pouvez récupérer les lignes en erreur, si vous le souhaitez. Pour cela, utilisez un lien Row > Rejects.

 

Schema et Edit schema

Un schéma est une description de lignes, il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (Built-in), soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma. Notez que si vous effectuez des modifications, le schéma devient automatiquement built-in.

 

 

Built-in : Sélectionnez cette option pour éditer le schéma Built-in pour les données à traiter.

 

 

Repository : Sélectionnez cette option pour éditer le schéma du Repository sélectionné. Les champs suivants sont automatiquement renseignés à l'aide du schéma sélectionné..

 

Edit schema

Cliquez sur le bouton [...] pour éditer le schéma Built-in ou Repository pour les données à traiter.

 

Sync columns

Cliquez sur ce bouton pour récupérer le schéma défini dans le composant d'entrée.

Note

Ce bouton est disponible uniquement lorsqu'un composant d'entrée est connecté à ce composant via un lien Row > Main.

 Advanced settings

Encoding

Sélectionnez l’encodage à partir de la liste ou sélectionnez Custom et définissez-le manuellement. Ce champ est obligatoire pour la manipulation des données de base de données.

 

Trim all column

Cochez cette case pour supprimer les espaces en début et en fin de champ dans les colonnes définies.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du Job ainsi qu'au niveau de chaque composant.

Utilisation

Ce composant gère un flux de données et nécessite donc un composant d'entrée et un composant de sortie. Cela vous permet d'extraire des données d'un fichier délimité, à l'aide d'un lien Row > Main et vous permet de créer un flux de rejet filtrant des données dont le type ne correspond pas au type défini.

Scénario

Pour un scénario associé, consultez la section intitulée « Scénario : Extraire des noms, des domaines et domaines de premier niveau à partir d’adresses e-mail ».
