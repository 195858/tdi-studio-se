Scénario : Exécuter un job distant

Le scénario suivant décrit un job à un composant appelant et exécutant un autre job. Le job à exécuter lit un fichier délimité de base et affiche son contenu dans la console de la vue Run. La particularité de ce job réside dans le fait que ce dernier est exécuté à partir d'un autre job et utilise la variable de contexte provenant du fichier d'entrée devant être traité.

Créez le premier job lisant le fichier délimité.

    • Cliquez et déposez les composants tFileInputDelimited et tLogRow dans l'espace de modélisation.
    • Dans l'onglet Basic settings de la vue Component du tFileInputDelimited, paramétrez les propriétés du composant d'entrée.
    • Dans la liste déroulante Property type, sélectionnez l'option Built-In.
    • Dans le champ File Name, renseignez le chemin d'accès au fichier d'entrée. Dans cet exemple, ce fichier est le fichier Comprehensive.txt.
    • Sélectionnez ce chemin d'accès et appuyez sur la touche F5 pour ouvrir la fenêtre de configuration de variable.
    • Nommez cette nouvelle variable de contexte, ici appelez-la File.
    • Pour cet exemple, laissez les paramètres par défaut, i.e. ne cochez pas la case Prompt for value et laissez le champ Prompt vide.
    • Cliquez sur Finish pour valider et appuyez sur Enter pour vous assurer que la nouvelle variable de contexte est conservée dans le champ File Name.
    • Dans les champs Field et Row separators de l'onglet Basic settings, renseignez les séparateurs de champs et de lignes utilisés dans le fichier d'entrée.
    • Dans cet exemple, le fichier ne possède ni d'en-tête ni de pied de page. Ainsi, laissez les champs Header et Footer vides. Mais dans le Limit, paramétrez le nombre de lignes à traiter à 50.
    • Dans la liste déroulante Schema type, sélectionnez l'option Built-in. Cliquez sur le bouton [...] pour configurer manuellement le schéma.
    • Ajoutez deux colonnes et nommez-les comme les deux premières colonnes du fichier d'entrée. Dans cet exemple : ID et Registration.
    • Si vous avez stocké le schéma dans le Repository, sélectionnez la métadonnées correspondant à la structure de votre fichier d'entrée.
    • Puis connectez votre composant d'entrée au composant de sortie : le tLogRow.

Créez le deuxième job jouant le rôle d'un job maître.

    • Dans la vue Component du composant tRunJob, sélectionnez le job à exécuter.
    • Nous vous conseillons d'éxécuter une fois le job appelé avant de l'exécuter via le composant tRunJob pour vous assurez qu'il s'exécute correctement.
    • Dans le champ Context, sélectionnez le contexte adéquat. Pour cet exemple, seul le contexte Default est disponible et contient la variable de contexte créée auparavant.
    • Cliquez sur le bouton Generate Code pour valider le contexte sélectionné et généré le code correspondant.
    • Enregistrez le job maître et appuyez sur la touche F6 pour l'exécuter.

Le job appelé lit les données contenues dans le fichier d'entrée comme défini dans le schéma d'entrée et le résultat de ce job est directement affiché dans la console Run.
