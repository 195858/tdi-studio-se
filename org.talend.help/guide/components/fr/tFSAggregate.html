tFSAggregate

Avertissement

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition de Talend Enterprise Data Integration Big Data edition. Pour plus d’informations concernant le mode Massively Parallel eXtension, consultez le Guide utilisateur de Talend Enterprise Data Integration Big Data edition.

Propriétés du tFSAggregate

Famille de composant

FileScale

 

Fonction

Le composant tFSAggregate effectue une agrégation basée sur une ou plusieurs colonnes. Pour chacune des lignes en sortie, une clé d’agrégation est fournie, ainsi que le résultat de l’opération d’agrégation correspondant (count, avg et sum).

Objectif

tFSAggregate permet d’établir des métriques et des statistiques basées sur des valeurs ou des calculs.

Basic settings

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma.

Note

Si vous modifiez le schéma, il devient automatiquement built-in.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et jobs. Voir également le Guide utilisateur de Talend Enterprise Studio.

 

 

Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également le Guide utilisateur de Talend Enterprise Studio.

 

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Input File Name

Nom du fichier contenant les données que vous souhaitez agréger.

 

Output File Name

Nom du fichier dans lequel vous souhaitez écrire les données agrégées.

 

Record separator (char)

Caractère, chaîne ou expression régulière séparant les lignes.

 

Field separator (char)

Caractère, chaîne ou expression régulière séparant les champs.

 

Header

Nombre de lignes d’en-tête à ignorer en début de fichier.

 

Footer

Nombre de lignes à ignorer en fin de fichier.

 

Group by

Column : Liste des colonnes du fichier d’entrée.

Key Attribute : Cochez la case correspondant au nom de la colonne à utiliser pour regrouper les données.

 

Operations

Additional Output Column : Saisissez le nom que vous souhaitez donner à la colonne contenant le résultat de l’opération d’agrégation.

 

 

Function : Sélectionnez le type d’opérations d’agrégation à effectuer.

count : calcule le nombre de lignes,

avg : calcule la moyenne,

sum : calcule la somme.

 

 

Input Column : Sélectionnez la colonne d’entrée à partir de laquelle les valeurs sont collectées pour l’agrégation.

Advanced settings

Generate FSLang File

Cochez cette case pour générer le fichier FSLang correspondant à votre Job et cliquez sur le bouton [...] à côté du champ FSLang File Name pour spécifier son chemin d’accès.

 

Assign FileScale Path

Cochez cette case puis cliquez sur le bouton [...] à côté du champ FileScale Path pour sélectionner le fichier exécutable correspondant au programme FileScale permettant d’exécuter le composant.

 

Sort results

Cochez cette case afin de trier les résultats.

 

Custom FileScale Parameter (separated by ,)

Saisissez les paramètres pour chaque opération spécifique que vous souhaitez ajouter à l’appel de l’exécutable FileScale.

 

Specify Number of Process Child

Cochez cette case et saisissez le nombre de processus fils à utiliser pour l’exécution de l’agrégation.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Utilisation

Ce composant traite des fichiers, ainsi il ne nécessite pas de flux de données en entrée ou en sortie. Il est utilisé pour agréger les données de fichiers volumineux.

Limitation

La limitation dépend des limites de la mémoire physique et de l’architecture du processeur. Par exemple, la taille totale des fichiers traités ne peut pas excéder la limite du système de fichiers pour supporter les fichiers très volumineux (maximum de la valeur 64 bits signée).

Scénario : Agréger les données d’un fichier volumineux

Ce scénario a été créé en Java et décrit un Job agrégant, très rapidement, un fichier volumineux contenant des données clients en fonction de l’Etat dans lequel ils sont basés, calculant le nombre de clients ayant été agrégés pour chaque Etat et faisant une moyenne de leurs bénéfices grâce au composant tFSAggregate.

Avertissement

Assurez-vous d’avoir bien enregistré localement le fichier exécutable correspondant à votre système d’exploitation afin de pouvoir indiquer son chemin d’accès dans Talend Enterprise Studio.

Dans ce scénario, vous avez déjà centralisé le schéma du fichier dans le Repository. Pour plus d’informations concernant la création et l’utilisation de métadonnées de schémas dans le Repository, consultez le Guide utilisateur de Talend Enterprise Studio.

Le fichier contient neuf colonnes : id, CustomerName, CustomerAddress, idState, id2, RegTime, RegisterTime, Sum1 et Sum2.

  • Dans le Repository, développez le nœud Metadata et le nœud de la métadonnée que vous venez de créer contenant le schéma d’entrée et glissez ce nœud dans le Job designer.

    La boîte de dialogue [Components] apparaît.

  • Sélectionnez le composant tFSAggregate dans la liste et cliquez sur OK pour fermer la boîte de dialogue.

    Le composant tFSAggregate apparaît dans le Job designer.

  • Double-cliquez sur le composant tFSAggregate pour paramétrer ses propriétés dans l’onglet Basic settings de la vue Component.

Tous les champs de propriétés du tFSAggregate sont automatiquement renseignés. Si vous n’avez pas défini le schéma dans les métadonnées du Repository, renseignez manuellement ces informations après avoir sélectionné le mode Built-in dans les listes Schema Type et Property Type.

  • Dans le champ Output File Name, indiquez le nom et le chemin d’accès du fichier de sortie contenant les données agrégées.

  • Dans la zone Group by, cochez la case correspondant à la colonne sur laquelle effectuer l’opération de regroupement. Vous pouvez sélectionner plusieurs colonnes afin d’effectuer des regroupements sur plusieurs critères. Pour ce scénario, regroupez les données de la colonne idState.

  • Dans la zone Operations, cliquez sur le bouton [+] pour ajouter deux colonnes contenant le résultat des opérations d’agrégation.

  • Dans le champ Additional Output column de la première ligne, saisissez un nom pour la première colonne supplémentaire, count pour ce scénario.

  • Dans le champ Function, sélectionnez l’opération d’agrégation que vous souhaitez exécuter, count pour ce scénario.

  • Dans le champ Input Column, sélectionnez la colonne dont la valeur d’entrée sera récupérée pour l’opération d’agrégation, id pour ce scénario.

Ainsi, la colonne count sera ajoutée au fichier de sortie et contiendra le nombre d’id des clients regroupés par Etat.

  • Dans le champ Additional Output column de la deuxième ligne, saisissez le nom de la deuxième colonne supplémentaire, avg.

  • Dans le champ Function, sélectionnez l’opération d’agrégation que vous souhaitez exécuter, avg pour ce scénario.

  • Dans le champ Input Column, sélectionnez la colonne dont la valeur d’entrée sera récupérée pour l’opération d’agrégation, Sum1 pour ce scénario.

Ainsi, la colonne avg sera ajoutée au fichier de sortie et fera la moyenne des Sum1 des clients par Etat.

  • Cliquez sur l’onglet Advanced settings pour paramétrer les propriétés avancées du composant et cochez la case Assign FileScale Path et dans le champ FileScale Path, indiquez le chemin d’accès au fichier exécutable correspondant à votre système d’exploitation.

  • Enregistrez le Job et appuyez sur F6 pour l’exécuter.

Une barre de progression apparaît sous le composant tFSAggregate dans le Job designer pour montrer le pourcentage de progression de l’opération d’agrégation. Cette barre de progression vous montrera la rapidité de l’exécution de l’agrégation.

Lorsque le pourcentage de progression atteint les 100%, les données sélectionnées sont regroupées et transmises en sortie, ainsi que les deux nouvelles colonnes contenant le compte des id des données regroupées et la moyenne des Sum1.
