Scénario : Créer un index de synonymes

Ce scénario décrit un Job à trois composants permettant de créer un index des noms de villes standardisés, capable de fournir les références des synonymes des villes utilisés dans les données clients d'une entreprise.

Afin de créer cet index, un fichier source est nécessaire pour fournir le nom des villes et leurs synonymes associés. Pour ce scénario, utilisez un fichier .csv comme suit :

Ce fichier contient deux colonnes :

    • la colonne de gauche est la colonne CityName qui contient le nom des villes standardisés comme données de référence.
    • la colonne de droite est la colonne Synonyms qui contient plusieurs synonymes provenant des données clients de cette entreprise.

Les trois composants utilisés pour ce scénario sont :

    • le tFileInputDelimited : ce composant charge des données à partir d'un fichier source et les entre dans le tSynonymOutput.
    • le tSynonymOutput : ce composant crée l'index utilisé pour ce scénario et le remplit avec les synonymes définis dans le fichier source.
    • le tLogRow : ce composant liste les données qui ont été insérées dans l'index nouvellement créé.

Pour reproduire ce scénario, procédez comme suit :

    • A partir de la Palette, déposez les composants suivants dans l'espace de modélisation graphique : un tFileInputDelimited, un tSynonymOutput et un tLogRow.



      • Au besoin, vous pouvez changer le nom de chaque composant, comme cela a été fait dans ce scénario pour le composant tFileInputDelimited, renommé CityNames. Pour plus d'informations, consultez la section Onglet View du Guide utilisateur Talend Integration Suite Studio.

    • Cliquez-droit sur le composant tFileInputDelimited (CityNames), et connectez-le au composant tSynonymOutput à l'aide d'un lien Row > Main.
    • De la même façon, connectez le tSynonymOutput au tLogRow à l'aide d'un lien Row > Main.
    • Double-cliquez sur le composant tFileInputDelimited (CityNames) pour ouvrir l'onglet Basic settings de sa vue Component.
    • Dans le champ File name/Stream, saisissez le chemin d'accès, ou parcourez vos répertoires et sélectionnez l'emplacement où vous souhaitez créer l'index.
    • A côté du champ Schema, cliquez sur le bouton Edit schema pour ouvrir l'éditeur de schéma.
    • Dans cet éditeur, cliquez sur le bouton [+] à deux reprises pour ajouter deux lignes.
    • Dans la colonne Column, cliquez sur chaque nouvelle ligne et renommez-les respectivement : CityName et Synonyms. Ces noms sont conformes avec les noms des colonnes du fichier source.
    • Cliquez sur OK pour valider ces changements et acceptez leur propagation dans la boîte de dialogue qui apparaît.



      • Vous avez également la possibilité d'ajouter ce schéma en utilisant les métadonnées stockées dans le Repository. Cela vous permet d'utiliser automatiquement la configuration des métadonnées correspondantes. Pour plus d'informations concernant la création et l'utilisation des métadonnées, consultez Configurer un schéma File Delimited dans le Guide utilisateur Talend Integration Suite Studio.

    • Double-cliquez sur le composant tSynonymOutput pour ouvrir l'onglet Basic settings de sa vue Component.
    • Dans le champ Index path, saisissez le chemin d'accès, ou parcourez votre répertoire et sélectionnez l'emplacement où vous souhaitez créer l'index. Au moment d'exécuter votre Job, le répertoire à utiliser pour contenir l'index créé doit exister.
    • Dans le champ Operation, sélectionnez l'opération à effectuer sur l'index créé ainsi que sur les synonymes associés. Pour ce scénario, sélectionnez Initialize an index and insert documents.
    • Dans le champ Entry, sélectionnez la colonne à utiliser pour recevoir et stocker les données de référence standardisées. Dans le fichier source utilisé pour ce scénario, sélectionnez la colonne CityName qui contient les noms de villes standards.
    • Dans le champ Synonyms, sélectionez la colonne à utiliser pour recevoir et stocker les synonymes. Pour ce scénario, sélectionnez Synonyms.
    • Appuyez sur F6 pour exécuter ce Job.

Les noms des villes standardisés ainsi que leurs synonymes insérés dans l'index nouvellement créé sont affichés dans la console de la vue Run.
