Scénario : Extraire uniquement les données qui correspondent à un modèle défini d'un fichier délimité

Ce scénario Java décrit un Job à trois composants, dans lequel le composant tExtractPattern est utilisé afin d'extraire uniquement les adresses e-mail des clients (qui correspondent au modèle Email address) d'un fichier délimité contenant différentes données sur les clients. Il écrit ensuite les données extraites dans un autre fichier délimité.

    • Glissez les composants suivants de la Palette dans l'espace de modélisation graphique : tFileInputDelimited, tPatternExtract et tFileOutputDelimited.
    • Double-cliquez sur le tFileInputDelimited afin d'afficher sa vue Basic settings et définir ses propriétés. Si vous avez déjà stocké vos schémas d'entrée dans le Repository, vous pouvez simplement glisser la métadonnée correspondante du noeud Metadata dans l'espace de modélisation graphique. Ainsi, les paramètres seront automatiquement renseignés, dans la vue Basic settings. Pour plus d'informations sur les métadonnées, consultez le Guide Utilisateur de Talend Integration Suite Studio.

Dans ce scénario, le fichier délimité contient les noms, les adresses e-mail et les numéros de téléphone dans une seule colonne : Name_Telephone_Address.

Le schéma d'entrée du fichier délimité sera défini comme suit :

    • Si vous n'avez pas stocké les schémas des fichiers délimités dans le Repository, définissez manuellement les propriétés du composant, dans la vue Basic settings, et cliquez sur Edit schema afin d'afficher la boîte de dialogue où vous pourrez définir le schéma d'entrée.
    • Cliquez sur le bouton [+], afin d'ajouter autant de lignes que vous avez dans votre schéma d'entrée, une seule dans ce scénario, puis cliquez sur OK pour fermer la boîte de dialogue.



      • Une nouvelle boîte de dialogue s'ouvre et vous demande si vous voulez propager les changements, cliquez sur OK si vous souhaitez accepter la propagation du schéma. Ainsi, les composants suivants obtiendront automatiquement le schéma d'entrée. Il n'y a donc pas besoin de paramétrer le schéma à nouveau.

    • Double-cliquez sur le tPatternExtract afin d'afficher sa vue Basic settings et définir ses propriétés.
    • Dans la liste Column to check, sélectionnez la colonne dont vous voulez vérifier les données, par rapport au modèle défini, Name_Telephone_Address dans cet exemple.
    • Dans la liste Pattern type, sélectionnez le modèle en accord avec lequel vous souhaitez extraire les données, Email address dans cet exemple.
    • Cliquez sur le bouton Sync columns afin de propager automatiquement toutes les colonnes du fichier délimité d'entrée.
    • Au besoin, cliquez sur Edit schema afin de voir la structure des données des composants tFileInputDelimited et tPatternExtract.
    • Cliquez sur OK pour fermer la boîte de dialogue.
    • Double-cliquez sur le tFileOutputDelimited afin d'afficher sa vue Basic settings et définir ses propriétés.
    • Cliquez sur le bouton [...] à côté du champ File Name, afin de sélectionner le fichier dans lequel vous souhaitez écrire vos données de sortie.
    • Définissez les séparateurs de lignes et de champs, au besoin. Dans cet exemple, l'objectif est de séparer les adresses e-mail des clients par un point virgule.
    • Cochez la case Append si vous voulez ajouter de nouvelles lignes à la fin des enregistrements.
    • Cochez la case Include Header pour inclure le titre de la colonne dans les données de sortie.
    • Au besoin, cliquez sur Edit schema afin de voir les flux de données d'entrée et de sortie.



      • Notez que le schéma dans les composants de sortie est différent du schéma du composant qui les précède, puisque vous voulez extraire uniquement les données qui correspondent au modèle Email address.

    • Sauvegardez votre Job et appuyez sur F6 pour l'exécuter.

Les adresses e-mail des clients sont extraites de la colonne sélectionnée, selon le modèle Email défini, et écrites dans le fichier de sortie, séparées par un point virgule. Vous pouvez, par exemple, envoyer un mail à tous vos clients en une seule fois.
