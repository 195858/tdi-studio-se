tFSFilterColumns

Avertissement

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition de Talend Big Data Studio. Pour plus d’informations concernant le mode Massively Parallel eXtension, consultez le Guide Utilisateur de Talend Big Data Studio.

Propriétés du tFSFilterColumns

Famille de composant

FileScale

 

Fonction

Le composant tFSFilterColumns apporte des changements spécifiques à la table sélectionnée en fonction du mapping effectué sur les noms de colonnes. Ce composant permet de filtrer très rapidement les colonnes des fichiers volumineux. Pour optimiser les performances, le composant trie généralement les données avant de les traiter.

Objectif

Ce composant permet d’homogénéiser les structures d’un fichier en supprimant les colonnes superflues ou en ajoutant de nouvelles colonnes.

Basic settings

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma.

Note

Si vous modifiez le schéma, il se met au­tomatiquement en built-in.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et job designs. Voir également le Guide utilisateur de Talend Data Integration Studio.

 

 

Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également le Guide utilisateur de Talend Data Integration Studio.

 

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Input File Name

Nom du fichier contenant les données que vous souhaitez filtrer.

 

Output File Name

Nom du fichier dans lequel vous souhaitez écrire les données filtrées.

 

Record separator (char)

Caractère, chaîne ou expression régulière séparant les lignes.

 

Field separator (char)

Caractère, chaîne ou expression régulière séparant les champs.

 

Header

Nombre de lignes d’en-tête à ignorer en début de fichier.

 

Footer

Nombre de lignes à ignorer en fin de fichier.

 

Column configuration

Column : Liste des colonnes à filtrer.

Keep it : Cochez la case correspondant au nom de la colonne que vous souhaitez maintenir dans le fichier. Si vous décochez la case, la colonne sera supprimée.

Saisissez les paramètres pour chaque opération spécifique que vous souhaitez ajouter à l’appel de l’exécutable FileScale.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Utilisation

Ce composant traite des fichiers, ainsi il ne nécessite pas de flux de données en entrée ou en sortie. Il sert à vérifier les données contenues dans des fichiers volumineux.

Limitation

La limitation dépend des limites de la mémoire physique et de l’architecture du processeur. Par exemple, la taille totale des fichiers traités ne peut pas excéder la limite du système de fichiers pour supporter les fichiers très volumineux (maximum de la valeur 64 bits signée).

Scénario : Regrouper les processus de filtrage et de tri dans un fichier à gros volume

Avertissement

Vérifiez que vous avez dézippé et sauvegardé localement le fichier exécutable FileScale fourni par Talend. Vous devez définir le chemin d’accès à ce fichier exécutable dans la vue Advanced settings du composant tFSJoin.

Ce scénario Java décrit un Job utilisant plusieurs composants FileScale connectés via le lien FS Combine. A l’aide de ce lien, les différents composants FileScale traiteront des données simultanément et optimiseront ainsi les performances. Ce scénario Java utilise les composants FileScale suivants :

  • tFSFilterColumns : filtre des colonnes dans une table, à partir du mapping du nom des colonnes,

  • tFSFilterRows : définit un filtre sur les lignes de la table, pour ne garder que les lignes dont la valeur est supérieure ou égale à 5000.0,

  • tFSUnique : récupère une seule occurrence de l’enregistrement name, même en cas de doublons,

  • tFSSort : trie les données par ordre alphabétique, selon la colonne name.

Afin de reproduire ce scénario, procédez comme suit :

  • Déposez les composants suivants de la Palette dans l’espace de modélisation graphique : tRowGenerator, tFileOutputDelimited, tFSFilterColumns, tFSFilterRows, tFSUnique et tFSSort.

  • Reliez le tRowGenerator au tFileOutputDelimited à l’aide d’un lien Row > Main puis au tFSFilterColumns à l’aide d’un lien OnSubjobOk.

  • Reliez les composants FileScale à l’aide de liens Row > FS Combine.

Note

Puisque le lien FS Combine groupe tous les processus de données créés dans le Job et les exécute simultanément, l’ordre des composants FileScale n’a aucune importance. Vous pouvez les disposer dans l’ordre que vous voulez.

Avertissement

Pour que ce scénario soit exécuté correctement, vous devez définir les mêmes fichiers d’entrée et de sortie dans tous les composants FileScale.

Dans ce scénario, le composant tRowGenerator va générer des données selon le schéma que vous avez défini dans la vue Basic settings et les envoyer au fichier d’entrée. Si la génération s’effectue sans erreur, les données du fichier d’entrée seront traitées simultanément dans les quatre composants FileScale, ce qui permet d’optimiser les performances.

  • Cliquez sur le tRowGenerator afin d’afficher sa vue Basic settings et définir ses propriétés.

  • Cliquez sur le bouton [...] à côté du RowGenerator Editor afin d’ouvrir l’éditeur du composant, dans lequel vous pouvez définir votre schéma.

  • Dans la moitié supérieure de l’éditeur, cliquez sur le bouton [+] pour ajouter les colonnes que vous souhaitez écrire dans le fichier d’entrée.

  • Définissez le schéma et configurez les paramètres des colonnes.

    Dans ce scénario, le fichier d’entrée contient neuf colonnes : id, name, dateOfBirth, amount et cinq autres colonnes contenant des informations supplémentaires.

  • Si nécessaire, cliquez sur l’onglet Preview dans la partie inférieure de l’éditeur pour afficher la vue correspondante puis cliquez sur le bouton View afin d’afficher une partie des données générées.

  • Cliquez sur OK pour valider votre schéma et fermer l’éditeur tRowGenerator.

  • Cliquez sur le composant tFileOutputDelimited pour afficher sa vue Basic settings et définir ses propriétés.

  • Configurez les propriétés du tFileOutputDelimited.

Dans ce scénario, définissez deux variables de contexte : filename pour le nom du fichier d’entrée et folder pour le chemin d’accès au fichier d’entrée. Pour plus d’informations concernant les variables de contexte, consultez le Guide Utilisateur Talend Data Integration Studio.

  • Placez votre souris sur le champ File Name, appuyez sur Ctrl+Espace et sélectionnez la variable de contexte pour le chemin d’accès au fichier (context.folder).

  • Saisissez +”/”+ pour séparer le chemin d’accès au fichier du nom du fichier puis appuyez sur Ctrl+Espace pour afficher la liste des variables de contexte.

  • Sélectionnez la variable de contexte pour le nom du fichier (context.filename).

  • Si nécessaire, cliquez sur le bouton Edit schema pour voir le schéma que vous avez défini dans l’éditeur et le modifier.

  • Cliquez sur le tFSFilterColumns afin d’ouvrir sa vue Basic settings et définir ses propriétés.

  • Dans les listes Schema et Property Type, sélectionnez Built-In.

  • Cliquez sur le bouton Edit schema pour ouvrir une boîte de dialogue. Vous pouvez définir les colonnes de votre schéma. Ce schéma doit correspondre au schéma du fichier d’entrée.

  • Cliquez sur OK pour fermer la boîte de dialogue du schéma.

    Les colonnes du schéma s’affichent dans le tableau Column configuration.

  • Configurez le nom du fichier d’entrée et son chemin d’accès en utilisant les variables de contexte définies précédemment en appuyant sur Ctrl+Espace et en sélectionnant la variable dans la liste. Séparez les deux variables par +”/”+.

  • Cliquez sur le bouton [+] à côté du champ Output File Name et parcourez votre répertoire jusqu’au fichier de sortie dans lequel vous souhaitez écrire les données traitées.

    Dans cet exemple, vous avez défini une variable de contexte pour le fichier de sortie, de la même façon que pour le fichier d’entrée.

  • Dans les champs Record Separator et Field Separator, saisissez respectivement le séparateur d’enregistrements puis le séparateur de champs, puis dans les champs Header et Footer, configurez respectivement les en-têtes et les pieds de page, s’il y en a.

  • Dans le tableau Column configuration, cochez la case des colonnes que vous souhaitez écrire dans le fichier de sortie, id, name, dateOfBirth et ce scénario.

  • Cliquez sur l’onglet Advanced settings pour afficher la vue correspondante.

  • Cochez la case Generate FSLang File pour afficher le champ FSLang File Name et parcourir votre répertoire jusqu’à l’emplacement dans lequel vous souhaitez générer le fichier FSLang correspondant à votre Job.

    Dans cet exemple, vous avez défini une variable de contexte pour le fichier de sortie FSLang comme vous l’avez fait pour les fichiers d’entrée et de sortie.

  • Cochez la case Assign FileScale Path pour afficher le champ FileScale Path puis parcourir votre répertoire jusqu’au fichier FileScale exécutable livré par Talend.

  • Cliquez sur le tFSFilterRows afin d’afficher sa vue Basic settings et définir ses propriétés.

  • Cliquez sur le bouton Edit schema pour ouvrir la boîte de dialogue du schéma dans laquelle vous pouvez définir les colonnes de votre schéma. Ce schéma doit correspondre au schéma du fichier d’entrée.

  • Dans le panneau de droite, supprimez la colonne de sortie ne correspondant pas au filtre sur les colonnes que vous avez défini dans le tFSFilterColumns, toutes les colonnes extraInfo dans ce scénario.

  • Cliquez sur OK pour fermer la boîte de dialogue du schéma.

  • Configurez les propriétés des composants comme pour le tFSFilterColumns. Utilisez les mêmes fichiers d’entrée et de sortie.

  • Sélectionnez un opérateur logique dans la liste correspondante, And dans ce scénario.

  • Dans le tableau Conditions, cliquez sur le bouton [+] pour ajouter une ligne.

  • Cliquez dans la ligne et sélectionnez la colonne sur laquelle vous souhaitez baser votre filtr sur les lignes.

    Dans ce scénario, vous allez écrire uniquement les lignes dans lesquelles la valeur amount est supérieure ou égale à 5000.0.

  • Cliquez sur l’onglet Advanced settings pour afficher la vue correspondante.

  • Cochez la case Assign FileScale Path pour afficher le champ FileScale Path puis parcourez votre répertoire jusqu’au fichier FileScale exécutable livré par Talend.

  • Cliquez sur le tFSUnique afin d’afficher sa vue Basic settings et définir ses propriétés.

  • Définissez les propriétés comme vous l’avez fait pour le tFSFilterRows. Utilisez les mêmes fichiers d’entrée et de sortie.

  • Dans le tableau Unique key, cochez la case de la colonne que vous souhaitez utiliser comme attribut de clé.

    Dans ce scénario, vous souhaitez récupérer une seule occurrence de chaque nom dans la colonne name.

  • Si vous souhaitez prendre en compte les majuscules et les minuscules, cochez la case Case Sensitive.

  • Cliquez sur l’onglet Advanced settings pour afficher la vue correspondante.

  • Cochez la case Assign FileScale Path pour afficher le champ FileScale Path et parcourir votre répertoire jusqu’au fichier FileScale exécutable livré par Talend.

  • Cliquez sur le tFSSort pour afficher sa vue Basic settings et définir ses propriétés.

  • Configurez les propriétés du composant comme vous l’avez fait avec le tFSUnique. Utilisez les mêmes fichiers d’entrée et de sortie.

  • Dans le tableau Criteria, cliquez sur le bouton [+] afin d’ajouter une ligne.

  • Cliquez dans le champ Schema column puis sélectionnez la colonne sur laquelle vous souhaitez vous baser pour trier les données.

  • Cliquez dans la colonne Sort type et sélectionnez le type de tri.

  • Cliquez dans la colonne Order type et sélectionnez l’ordre de tri.

Dans ce scénario, triez les données de sortie alphabétiquement en ordre ascendant basé sur la colonne name.

  • Cliquez sur l’onglet Advanced settings pour afficher la vue correspondante.

  • Cochez la case Assign FileScale Path pour afficher le champ FileScale Path puis parcourez votre répertoire jusqu’au fichier FileScale exécutable livré par Talend.

  • Sauvegardez votre Job et appuyez sur F6 pour l’exécuter.

Une barre de progression s’affiche dans l’espace de modélisation graphique et vous montre l’avancement des opérations simultanées. Cette barre de progression vous permet de voir combien de gros volumes de données d’entrée sont traités rapidement.

Lorsque le pourcentage atteint les 100%, les données sont écrites dans le fichier de sortie. Une simple comparaison entre les données d’entrée et les données traitées vous montre que le fichier de sortie n’a que les colonnes id, name, dateOfBirth et amount. Seules les lignes dans lesquelles la valeur amount est supérieure ou égale à 5000.0 sont affichées.

Une seule occurrence de chaque nom et de chaque enregistrement de données est écrite dans l’ordre alphabétique ascendant.
