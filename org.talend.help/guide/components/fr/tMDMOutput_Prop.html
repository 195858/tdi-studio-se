tMDMOutput


This component is available in the Palette , but you will only be able to use it if you have Talend MDM.

tMDMOutput properties

Component family
Talend MDM
Function
tMDMOutput writes data in an XML file in an MDM database.
Purpose
This component writes data in an MDM database.
Basic settings
Property Type
Either Built-in or Repository.
 
 
Built-in: No property data stored centrally
 
 
Repository: Select the repository file where the properties are stored. The fields which follow are filled in automatically using the fetched data.
 
Schema type and Edit Schema
A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository.
Click Edit Schema to modify the schema. Note that if you modify the schema, it automatically becomes built-in.
Click Sync columns to collect the schema from the previous component.
 
 
Built-in: You create the schema and store it locally for this component only. Related topic: Setting a built-in schema of Talend Open Studio User Guide.
 
 
Repository: You have already created the schema and stored it in the Repository. You can reuse it in various projects and job designs. Related topic: Setting a Repository schema in the Talend Open Studio User Guide.
 
XML Field
Select the name of the column in which you want to write the XML data.
 
URL
Type in the URL required to access the MDM database.
 
Username and Password
Type in the user authentication data for the MDM database.
 
Version
Type in the name of the perspective you want to connect to, for which you have the user rights required.
Leave this field empty if you want to display the default perspective.
 
Data Model
Type in the name of the data model on which the information to be written is based.
 
Data Container
Type in the name of the directory into which the data you want to write should be saved.
 
Is Update
Select this check box to update the modified fields.
If you leave this check box unchecked, all fields will be replaced by the modified ones.
 
With Report
Select this check box to add the actions carried out to a modification report.
Source Name: Between quotes, enter the name of the application to be used to carry out the modifications.
Enable verification by "before saving" transformer: Select this check box to verify the commit that has been just added; prior to saving.
 
Die on error
This check box is selected by default. Clear the check box to skip the row in error and complete the process for error-free rows. If needed, you can retrieve the rows in error via a Row > Rejects link.
Advanced settings
Extended Output
Select this check box to commit data in batches. You can specify the number of lines per batch in the Rows to commit field.
 
Configure Xml Tree
Opens the interface which helps create the XML structure to be written.
 
Group by
Select the column to be used to regroup the data.
 
Create empty element if needed
This check box is selected by default. If the content of the interface's Related Column which enables creation of the XML structure is null, or if no column is associated with the XML node, this option creates an opening and closing tag at the required places.
 
Advanced separator (for number)
Select this check box to modify the number separators used by default.
- Thousands separator: enter the thousands separator between inverted commas.
- Decimal separator: enter the decimal separator between inverted commas.
 
Generation mode
Select the generation mode you want to use, according to the memory you have available.
- Fast but memory consuming (Dom4J)
- Slow with no memory consumed
 
Encoding Type
Select the encoding type from the list or else select Custom and define it manually. This is an obligatory field for the manipulation of data in a database.
 
tStatCatcher Statistics
Select this check box to gather the processing metadata at the Job level as well as at each component level.
Usage
Use this component to write a file and separate the fields using a specific separator.
