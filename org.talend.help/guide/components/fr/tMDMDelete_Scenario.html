Scénario : Supprimer des données d'une base de données MDM

Ce scénario décrit un Job à deux composants supprimant les données spécifiées d'une base de données MDM.

    • Glissez-déposez de la Palette dans l'espace de modélisation graphique un composant tMDMInput et un composant tMDMDelete, de la famille Talend MDM.
    • Connectez les deux composants à l'aide d'un lien Row > Main.
    • Double-cliquez sur le composant tMDMInput afin d'afficher sa vue Component et définir ses propriétés.
    • Dans la liste Property Type, sélectionnez Built-in pour renseigner les champs manuellement. Si vous avez stocké votre connexion MDM dans les métadonnées du Repository, sélectionnez Repository dans la liste, et les champs seront automatiquement remplis.
    • Dans la liste Schema Type, sélectionnez Built-in et cliquez sur le bouton [...] à côté du champ Edit schema, afin d'ouvrir une boîte de dialogue dans laquelle vous pourrez dfinir la structure des données que vous souhaitez lire dans la base de données MDM.
    • Les données sont collectées dans un schéma composé de quatre colonnes de type String: Id, Name, City et State. Cliquez sur OK.
    • Dans le champ URL, saisissez entre guillemets l'URL de la base de données MDM : "http://localhost:8080/talend/TalendPort".
    • Dans les champs Username et Password, saisissez votre identifiant et votre mot de passe de connexion à la base de données MDM. Dans ce scénario, nous nous connectons avec l'utilisateur Frank.
    • Dans le champVersion, saisissez entre guillemets le nom de la perspective à laquelle vous souhaitez accéder. Laissez ce champ vide afin d'afficher la perspective par défaut.
    • Dans le champ Entity, saisissez entre guillemets le nom de l'entité contenant les données que vous souhaitez lire. Ici, nous travaillons avec l'entité Agency.
    • Dans le champ Data Container, saisissez entre guillemets le nom du répertoire contenant les données que vous voulez lire. Ici, nous utilisons le conteneur DStar.
    • La case Use multiple conditions est cochée par défaut. Saisissez une condition pour filtrer les données que vous souhaitez supprimer. Dans le tableau Operations, cliquez sur le bouton [+] afin d'ajouter une nouvelle ligne.
      Dans la colonne Xpath, saisissez entre guillemets le chemin Xpath et la balise du noeud XML sur lequel vous souhaitez appliquer le filtre. Ici, nous travaillons avec l'entité Agency, saisissez "Agency/Id".
      Dans la colonne Function, sélectionnez la fonction que vous souhaitez utiliser. Dans ce scénario, nous utilisons la fonction Starts With.
      Dans la colonne Value, saisissez la valeur de votre filtre. Ici, nous souhaitons filtrer les données dont l'ID commence par "TA".
    • Dans la vue Component, cliquez sur l'onglet Advanced settings afin de configurer les paramètres avancés.
    • Dans le champ Loop XPath query, saisissez entre guillemets la structure et le nom du noeud XML sur lequel exécuter la boucle.
    • Dans la zone Mapping et dans la colonne XPath query, saisissez entre guillemets le nom de la balise XML sur laquelle nous souhaitons collecter les données, à côté de la colonne de sortie correspondante.
    • Dans l'espace de modélisation graphique, cliquez sur le composant tMDMDelete afin d'afficher sa vue Component et définir ses propriétés.
    • Dans la liste Schema Type, sélectionnez Built-in et cliquez sur le bouton [...] à côté du champ Edit Schema, afin de décrire la structure des données que vous voulez charger dans la base de données MDM.
    • Cliquez sur le bouton [+], à droite, afin d'ajouter une colonne de type String. Nommez-la xmlOutput. Cliquez sur OK.
    • Dans le champ URL, saisissez l'URL requise pour vous connecter à la base de données MDM : "http://localhost:8080/talend/TalendPort".
    • Dans les champs Username et Password, saisissez les informations d'authentification requises afin de vous connecter à la base de données. Dans ce scénario, nous nous connectons avec l'utilisateur Frank.
    • Dans le champ Version, saisissez entre guillemets le nom de la perspective à laquelle vous souhaitez accéder. Laissez le champ vide si vous souhaitez accéder à la perspective par défaut.
    • Dans le champ Entity, saisissez le nom de l'entité contenant les données que vous souhaitez supprimer. Ici, nous travaillons avec l'entité Agency.
    • Dans le champ Data Container, saisissez le nom du répertoire contenant les informations à supprimer. Ici, nous utilisons le conteneur DStar.
    • Dans le tableau Keys, cliquez sur le bouton [+] afin d'ajouter une nouvelle ligne. Dans la colonne Keys, sélectionnez la colonne contenant la clé de l'entité Agency. Si l'entité possède une clé multiple, ajoutez autant de lignes qu'il y a de champs dans la clé, et sélectionnez-les dans l'odre de la séquence. Ici, la clé de l'entité Agency est dans le champ Id.
    • Cochez la case Logical delete afin d'envoyer les données supprimées dans la Corbeille, et remplissez le champ du chemin d'accès à la Corbeille (Recycle bin path). Ici, nous laissons le chemin d'accès par défaut, mais si votre Corbeille se trouve dans un dossier spécifique, renseignez le chemin d'accès. Une fois dans la Corbeille, les données peuvent être supprimées définitivement, ou bien restaurées. Si vous laissez cette case décochée, les données seront supprimées définitivement.
    • Appuyez sur Ctrl+S pour sauvegarder votre Job, puis sur F6 pour l'exécuter.

Les données dont l'ID commence par "TA" ont été supprimées et envoyées dans la Corbeille de la base de données MDM.
