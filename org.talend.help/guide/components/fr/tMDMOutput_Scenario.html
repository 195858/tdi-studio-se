Scenario: Writing data in an MDM database

This scenario describes a two-component Job that generates data, transforms it into XML and loads it into an MDM database.

    • From the Palette, drop tFixedFlowInput and tMDMOutput onto the design workspace
    • Connect the components using a Row Main link.
    • Double-click tFixedFlowInput to view its Basic settings, in the Component tab and set the component properties.
    • In the Schema Type list, select Built-in and click the three-dot button next to Edit schema to open a dialog box in which you can define the structure of the data you want to write in the MDM database.
    • Click the plus button and add three columns of the type String. Name the columns: ISO2Code, Name, Currency. Click OK.
    • Click OK to validate your changes and close the dialog box.
    • In the Number of rows field, enter the number of rows you want to generate.
    • In the Mode zone, select the mode Use Single Table to generate just one table.
    • In the Value fields, enter the values which correspond to each of the schema columns, between inverted commas.
    • In the design workspace, click tMDMOutput to open its Basic settings view in the Component tab and set the properties.
    • In the Property Type list, select Built-in and complete the fields manually. If you have saved the MDM connection information under Metadata in the repository, select Repository from the list and the fields which follow will be completed automatically.
    • In the Schema Type list, select Built-in and click on the three dot button next to the Edit Schema field to describe the structure of the data you want to load in the MDM database.
    • Click the plus button to the right to add one column of the type String. Name this column xmlOutput. Click OK.
    • The XML Field list is automatically filled in with the output column that you created.
    • In the URL field, enter the URL required to connect to the MDM database.
    • In the Username and Password fields, enter the authentication information required to connect to the database.
    • In the Version field, enter the name of the perspective you want to access, between inverted commas. Leave the field blank to access the default perspective.
    • In the Data Model field, enter the name of the model on which the information to be written is based, between inverted commas.
    • In the Data Container, enter the name of the directory into which the information is to be written and saved.
    • In the Component view, click on the Advanced settings tab to set the advanced parameters.
    • Click the three-dot button next to Configure Xml Tree to open the component editor.
    • In the Link target area to the right, click in the Xml Tree field.
    • Select rootTag and replace it with "Country".
    • In the Linker source area, select your three schema columns and drop them on the Country node.
      The [Selection] dialog box displays.
    • Select the Create as sub-element of target node option so that the three columns are linked to the three XML sub-elements of the Country node and then click OK to close the dialog box.
    • Right-click Currency, in the Link Target area and select Set as Loop Element in the contextual menu.
    • Click OK to validate your changes and close the dialog box.
    • Save your job and press F6 to run it.

The new country is created and is added to the MDM database.
