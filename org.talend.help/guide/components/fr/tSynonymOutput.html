tSynonymOutput

Avertissement

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition correspondante de Talend Enterprise Data Quality.

Propriétés du tSynonymOutput

Famille de composant

Data Quality

 

Fonction

Le composant tSynonymOutput crée un index de Lucene et le remplit avec les entrées et les synonymes qu’il reçoit.

Pour plus d'informations concernant l'accès et la gestion des mots et des entrées de référence (documents) d'un synonyme existant, à l'aide de l'éditeur d'index de synonymes, consultez le Guide utilisateur de Talend Enterprise Data Quality.

Pour plus d'informations concernant les index de synonymes disponibles, consultez l'annexe concernant les dictionnaires des synonymes de données dans le Guide utilisateur de Talend Enterprise Data Quality.

Note

Le calcul de similarité des synonymes est amélioré en version 5.1 du Studio. Si vos index ont été créés avec une version 5.0 ou antérieure et si vous devez les gérer à l'aide de cette méthode de calcul améliorée, vous devez mettre à jour ces index en exécutant le fichier IndexMigrator.jar téléchargeable à l'adresse : http://talendforge.org/svn/top/trunk/org.talend.dataquality.standardization.migration/dist/IndexMigrator.jar. La commande à utiliser pour exécuter ce fichier jar est 

java -jar IndexMigrator.jar <inputPath> <outputPath(optional)>

(où inputPath est le chemin d'entrée des jars à migrer et outputPath(optional) est l'emplacement de sortie facultatif des jars migrés).

Objectif

Le tSynonymOutput crée des index de synonymes auxquels certains composants comme le tStandardizeRow ou le tSynonymSearch peuvent renvoyer lors du traitement des données.

Basic settings

Schema et Edit schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

 

 

Built-in : Le schéma est créé et conservé ponctuellement pour ce composant seulement. Voir également le Guide utilisateur de Talend Enterprise Studio.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisable dans divers projets et Job designs. Voir également le Guide utilisateur de Talend Enterprise Studio.

 

Index path

Saisissez le chemin d’accès, ou parcourez vos répertoires et sélectionnez l’emplacement où vous souhaitez créer et stocker l’index de synonymes.

 

Operations

Sélectionnez l’opération à effectuer sur l’index spécifié dans le champ Index path.

Initialize an index and insert documents : cette opération crée un nouvel index ou vide un index existant puis le remplit avec les synonymes souhaités.

Insert new documents : cette opération insère les documents souhaités dans l’index existant. Les synonymes en doublon n’y sont pas insérés.

Update existing documents and insert if not existing : cette opération met à jour les synonymes existants et ajoute les nouveaux synonymes dans l’index.

Delete existing documents : cette opération supprime les documents existants dans l’index.

 

Entry

Sélectionnez la colonne que vous souhaitez insérer pour créer les entrées de l’index.

 

Synonyms

Sélectionnez la colonne que vous souhaitez insérer pour créer les synonymes correspondants aux différentes entrées d’index.

 

Synonym separator

Renseignez le séparateur à utiliser pour séparer les synonymes de chaque entrée d’index. Par défaut, ce séparateur est |.

Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les métadonnées de process du Job, aussi bien au niveau du Job qu’au niveau de chaque composant.

Connections

 

Liens de sortie (d’un composant à un autre) :

Row : Main; Reject

Trigger : Run if; On Component Ok; On Component Error.

Liens d’entrée (d’un composant à un autre) :

Row : Main; Reject

Pour plus d’informations concernant les connexions, consultez le Guide utilisateur Talend Enterprise Studio.

Utilisation

Ce composant requiert des données d’entrée provenant du composant précédent pour créer ou mettre à jour des index.

Scénario 1 : Créer un index des synonymes de noms de villes

Ce scénario décrit un Job à trois composants permettant de créer un index des noms de villes standardisés, capable de fournir les références des synonymes des villes utilisés dans les données clients d’une entreprise.

Afin de créer cet index, un fichier source est nécessaire pour fournir le nom des villes et leurs synonymes associés. Pour ce scénario, utilisez un fichier .csv comme suit :

Ce fichier contient deux colonnes :

  • la colonne de gauche est la colonne CityName qui contient le nom des villes standardisés comme données de référence.

  • la colonne de droite est la colonne Synonyms qui contient plusieurs synonymes provenant des données clients de cette entreprise.

Les trois composants utilisés pour ce scénario sont :

  • le tFileInputDelimited : ce composant charge des données à partir d’un fichier source et les entre dans le tSynonymOutput.

  • le tSynonymOutput : ce composant crée l’index utilisé pour ce scénario et le remplit avec les synonymes définis dans le fichier source.

  • le tLogRow : ce composant liste les données qui ont été insérées dans l’index nouvellement créé.

Pour reproduire ce scénario, procédez comme suit :

Déposer et relier les composants

  1. A partir de la Palette, déposez les composants suivants dans l’espace de modélisation graphique : un tFileInputDelimited, un tSynonymOutput et un tLogRow.

    Note

    Au besoin, vous pouvez changer le nom de chaque composant, comme cela a été fait dans ce scénario pour le composant tFileInputDelimited, renommé CityNames. Pour plus d’informations, consultez le Guide utilisateur de Talend Enterprise Studio.

    Cliquez-droit sur le composant tFileInputDelimited (CityNames), et connectez-le au composant tSynonymOutput à l’aide d’un lien Row > Main.

    De la même façon, connectez le tSynonymOutput au tLogRow à l’aide d’un lien Row > Main.

  2. Double-cliquez sur le composant tFileInputDelimited (CityNames) pour ouvrir l’onglet Basic settings de sa vue Component.

  3. Dans le champ File name/Stream, saisissez le chemin d’accès, ou parcourez vos répertoires et sélectionnez l’emplacement où vous souhaitez créer l’index.

  4. A côté du champ Schema, cliquez sur le bouton Edit schema pour ouvrir l’éditeur de schéma.

  5. Dans cet éditeur, cliquez sur le bouton [+] à deux reprises pour ajouter deux lignes.

  6. Dans la colonne Column, cliquez sur chaque nouvelle ligne et renommez-les respectivement : CityName et Synonyms. Ces noms sont conformes avec les noms des colonnes du fichier source.

    Note

    Vous avez également la possibilité d’ajouter ce schéma en utilisant les métadonnées stockées dans le Repository. Cela vous permet d’utiliser automatiquement la configuration des métadonnées correspondantes. Pour plus d’informations concernant la création et l’utilisation des métadonnées, consultez le Guide utilisateur de Talend Enterprise Studio.

  7. Cliquez sur OK pour valider ces changements et acceptez leur propagation dans la boîte de dialogue qui apparaît.

Configurer le composant tSynonymOutput

  • Double-cliquez sur le composant tSynonymOutput pour ouvrir l’onglet Basic settings de sa vue Component.

    Dans le champ Index path, saisissez le chemin d’accès, ou parcourez votre répertoire et sélectionnez l’emplacement où vous souhaitez créer l’index. Au moment d’exécuter votre Job, le répertoire à utiliser pour contenir l’index créé doit exister.

    Dans le champ Operation, sélectionnez l’opération à effectuer sur l’index créé ainsi que sur les synonymes associés. Pour ce scénario, sélectionnez Initialize an index and insert documents.

    Dans le champ Entry, sélectionnez la colonne à utiliser pour recevoir et stocker les données de référence standardisées. Dans le fichier source utilisé pour ce scénario, sélectionnez la colonne CityName qui contient les noms de villes standards.

    Dans le champ Synonyms, sélectionnez la colonne à utiliser pour recevoir et stocker les synonymes. Pour ce scénario, sélectionnez Synonyms.

Sauvegarder et exécuter le Job

  • Appuyez sur F6 pour exécuter ce Job.

Les noms des villes standardisés ainsi que leurs synonymes insérés dans l’index nouvellement créé sont affichés dans la console de la vue Run.

Scénario 2 : Créer un index des synonymes de noms de personnes en utilisant un tMap

Ce scénario décrit un Job à quatre composants permettant de créer un index contenant des noms de personnes et leurs diminutifs correspondants.

Les données source utilisées pour ce scénario sont stockées dans un fichier .csv et se présentent comme suit :

Cette figure présente uniquement une partie des entrées et des synonymes correspondants à insérer dans l’index en question. Ces données décrivent le pays d’origine des personnes (qui ne doivent pas être insérées dans l’index), leur prénom (entrées de référence) et les diminutifs (synonymes) souvent employés.

Les quatre composants utilisés pour ce Job sont :

  • le tFixedFlowInput : ce composant génère des lignes de données source et les intègre au tSynonymOutput.

  • le tMap : ce composant est utilisé pour transformer les données source en deux colonnes séparées représentant les prénoms et les diminutifs tout en ignorant les données relatives aux pays d’origine.

  • le tSynonymOutput : ce composant crée l’index utilisé pour ce scénario et le peuple avec les synonymes données dans le fichier source.

  • le tLogRow : ce composant liste les données insérées dans l’index nouvellement créé.

Pour reproduire ce scénario, procédez comme suit :

Déposer et relier les composants

  1. A partir de la Palette, déposez les composants suivants dans l’espace de modélisation graphique : un tFixedFlowInput, un tMap, un tSynonymOutput et un tLogRow.

    Note

    Vous pouvez modifier le nom de chaque composant. Pour plus d’informations, consultez le Guide utilisateur de Talend Enterprise Studio.

  2. Cliquez-droit sur le composant tFixedFlowInput pour ouvrir le menu contextuel, et connectez-le au composant tMap à l’aide d’un lien Row > Main.

  3. De la même façon, connectez le tMap au tSynonymOutput en utilisant une connexion de type Row > Main.

    Une boîte de dialogue s’affiche, vous invitant à nommer le lien que vous créez. Saisissez synonyms, par exemple, puis cliquez sur OK pour valider et fermer cette boîte de dialogue.

  4. Connectez le tSynonymOutput au tLogRow en utilisant une connexion de type Row > Main.

Configurer le flux d'entrée

  1. Double-cliquez sur le tFixedFlowInput pour ouvrir sa vue Component.

  2. Cliquez sur le bouton [...] à côté d’Edit schema pour ouvrir l’éditeur de schéma.

  3. Cliquez sur le bouton [+] pour ajouter le nombre de lignes correspondant aux données source utilisées. Pour ce scénario, ajoutez dix lignes.

  4. Dans la colonne Column, cliquez sur chaque ligne pour les renommer comme suit : ORINGIN, FIRSTNAMES, DIMINUTIVE, DIMINUTIVE1, DIMINUTIVE2, DIMINUTIVE3, DIMINUTIVE4, DIMINUTIVE5, DIMINUTIVE6, DIMINUTIVE7.

  5. Cliquez sur OK pour valider ces changements et fermer cet éditeur.

    Note

    Vous pouvez également ajouter ce fichier tFileInputDelimited en utilisant les métadonnées existantes qui sont stockées dans le Repository. Ceci vous permet d’utiliser automatiquement les configurations de ces métadonnées. Pour plus d’informations concernant la création et l’utilisation de métadonnées, consultez le Guide utilisateur de Talend Enterprise Studio.

    • Dans la zone Mode, sélectionnez l’option the Use Inline Content (delimited file) pour afficher ses champs correspondants.

    • Dans le champ Content, ajoutez les données source utilisées dans l’index de synonymes. Pour ce scénario, elles sont copiées et collées à partir d’un fichier .csv.

  7. Double-cliquez sur le tMap pour ouvrir son éditeur.

  8. En bas à droite (synonyms) de l’éditeur, cliquez sur le bouton [+] pour ajouter des lignes.

    Cliquez sur les lignes ajoutées dans la colonne Column pour les renommer respectivement : FirstName et Surname. Ainsi, les lignes FirstName et Surname sont ajoutées au tableau synonyms en haut à droite de l’éditeur.

    Dans la colonne Length de ces deux lignes, saisissez respectivement 140 et 300.

    En haut à gauche (du côté entrée), sélectionnez la ligne FIRSTNAMES et déposez-la dans la ligne FirstName de la colonne Expression du côté sortie (droite).

    Dans la ligne Surname de la colonne Expression du côté sortie (droite), saisissez DqStringHandling.safeConcat('|',).

    En haut à gauche (du côté entrée), sélectionnez les lignes de DIMINUTIVE à DMINUTIVE7 et déposez-les dans la ligne Surname de la colonne Expression afin que l’expression que vous avez saisie se présente comme suit : DqStringHandling.safeConcat('|',row1.DIMINUTIVE ,row1.DIMINUTIVE1,row1.DIMINUTIVE2,row1.DIMINUTIVE3, row1.DIMINUTIVE4,row1.DIMINUTIVE5,row1.DIMINUTIVE6 ,row1.DIMINUTIVE7)

  9. Cliquez sur OK pour valider ces modifications et acceptez la propagation que vous propose la boîte de dialogue qui s’affiche.

  10. Double-cliquez sur le tSynonymOutput pour ouvrir sa vue Component.

    • Dans le champ Index path, saisissez le chemin d’accès, ou parcourez votre système jusqu’à l’emplacement où vous souhaitez créer l’index. Lorsque vous exécutez votre Job, le dossier qui contient l’index doit exister.

    • Dans le champ Operation, sélectionnez l’opération que vous souhaitez effectuer sur l’index créé ainsi que sur les synonymes correspondants. Pour cet exemple, sélectionnez Initialize an index and insert documents.

    • Dans le champ Entry, sélectionnez la colonne qui recevra et stockera les entrées de référence. Pour ce scénario, il s’agit de la colonne FirstName.

    • Dans le champ Synonyms, sélectionnez la colonne qui recevra et stockera les synonymes. Pour ce scénario, il s’agit de la colonne Surname.

Sauvegarder et exécuter le Job

  • Appuyez sur F6 pour exécutez ce Job.

L’index est créé et vous pouvez visualiser son contenu dans la Console de la vue Run.
