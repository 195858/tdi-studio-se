tSAPBWInput

Propriétés du tSAPBWInput

Famille de composant

Business

 

Fonction

Le composant tSAPBWInput lit des données dans une base de données SAP BW à l'aide d'une connexion API JDBC et extrait des champs selon une requête SQL.

Objectif

Ce composant exécute une requête SQL dans un ordre strictement défini devant correspondre à la définition de votre schéma. Il passe ensuite la liste des champs au composant suivant via un lien Main > Row.

Basic settings

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier dans lequel sont stockées les propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (Built-in) , soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma. Notez que si vous effectuez des modifications, le schéma devient automatiquement built-in.

 

JDBC URL

Saisissez l’URL JDBC dans le champ associé pour vous connecter à la base de données. Par exemple, saisissez : jdbc:jdbc4olap://adresse IP/nom de la base de données pour vous connecter à une base de données SAP BW.

 

Username

Saisissez l'identifiant d'authentification à la base de données.

 

Password

Saisissez le mot de passe d'authentification à la base de données.

 

Table Name

Saisissez le nom de la table.

 

Query Type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier dans lequel sont stockées les propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Guess Query

Cliquez sur le bouton Guess Query pour générer la requête correspondant au schéma de votre table dans le champ Query.

 QuerySaisissez votre requête de base de données en faisant attention à ce que l’ordre des champs corresponde à celui défini dans le schéma.

Advanced settings

Trim all the String/Char columns

Cochez cette case pour supprimer les espaces en début et en fin de champ dans toutes les colonnes contenant des chaînes de caractères.

 

Trim column

Supprimez les espaces en début et en fin de champ dans les colonnes sélectionnées.

Note

Décochez la case Trim all the String/Char columns pour activer la case Trim columns.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Utilisation

Ce composant couvre toutes les possibilités de requête SQL dans les bases de données SAP BW via une connexion JDBC.

Limitation

n/a

Scénario : Lire des données d'une base de données SAP BW

Ce scénario décrit un Job comprenant deux composants, lisant des données d'une base de données SAP BW. Les données récupérées sont affichées dans la console.

Avant de configurer votre Job, assurez-vous que les prérequis suivants sont respectés :

  1. Vous devez copier les fichiers .jar suivants composant le pilote jdbc4olap dans le chemin d'accès à votre classe :

    - activation.jar

    - commons-codec.jar

    - jdbc4olap.jar

    - saaj-api.jar

    - saaj-impl.jar

  2. Assurez-vous d'avoir la dernière version du pilote jdbc4olap. Vous pouvez télécharger la dernière version du pilote jdbc4olap sur jdbc4olap download section (en anglais). Pour plus d'informations concernant l'utilisation du pilote jdbc4olap, consultez le Guide jdbc4olap User Guide (en anglais).

Dans ce scénario, les quatre opérations suivantes sont nécessaires :

  1. Configurez le Job.

  2. Configurez la connexion JDBC au serveur SAP BW.

  3. Configurez une requête.

  4. Affichez les données récupérées dans la console.

Configurer le Job

  1. Déposez un composant tSAPBWInput et un tLogRow de la Palette dans l'espace de modélisation graphique.

  2. Connectez le tSAPBWInput au tLogRow à l'aide d'un lien Main > Row.

Configurer la connexion JDBC au serveur SAP BW

  1. Double-cliquez sur le composant tSAPBWInput pour ouvrir sa vue Basic settings et configurer ses propriétés.

  2. Dans le champ JDBC URL, saisissez l'URL de votre serveur jdbc4olap.

    Notez que l'URL ci-dessus est uniquement affichée dans un but de démonstration.

  3. Dans les champs Username et Password, saisissez respectivement votre identifiant et votre mot de passe d'authentification à la base de données.

  4. Cliquez sur le bouton [...] à côté du champ Edit schema pour définir le schéma à utiliser.

  5. Cliquez sur le bouton [+] pour ajouter de nouvelles colonnes au schéma et définissez le type de données pour chaque colonne et cliquez sur OK afin de sauvegarder vos paramètres.

Configurer une requête

  1. Dans la vue Basic settings du composant tSAPBWInput, dans le champ Table Name, saisissez le nom de la table. Dans ce scénario, le nom de la table "Measures" est uniquement pour démonstration.

  2. Dans le champ Query, saisissez le script de la requête. Dans cet exemple, saisissez :

    "SELECT 
T1.\"[0D_CO_CODE].[LEVEL01]\" AS company, 
T0.\"[Measures].[D68EEPGGHUMSZ92PIJARDZ0KA]\" AS amount 
FROM 
\"0D_DECU\".\"0D_DECU/PRE_QRY4\".\"[Measures]\" T0, 
\"0D_DECU\".\"0D_DECU/PRE_QRY4\".\"[0D_CO_CODE]\" T1 "

    Note

    Les requêtes SQL supportées étant limitées, les scripts de requêtes à utiliser doivent être basés sur la grammaire définie dans le pilote jdbc4olap. Pour plus d'informations concernant cette grammaire, consultez le guide jdbc4olap User Guide (en anglais).

Afficher les données récupérées dans la console

  1. Double-cliquez sur le composant tLogRow pour ouvrir sa vue Basic settings et configurer ses propriétés.

  2. Cliquez sur le bouton Sync columns pour récupérer le schéma défini dans le composant précédent.

  3. Sélectionnez Table dans la zone Mode.

  4. Appuyez sur Ctrl+S pour sauvegarder votre Job et appuyez sur F6 pour l'exécuter.

    Les données de la table "Measure" sont récupérées et affichées dans la console.
