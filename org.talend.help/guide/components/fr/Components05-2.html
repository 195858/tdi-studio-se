A.1 Terminologie de base

Les termes de base utilisés dans FileScale sont :
· enregistrement : l'unité de base à trier. Dans les fichiers orientés ligne, l'enregistrement correspond à la ligne.
· champ (dans un enregistrement) : un enregistrement est constitué de plusieurs champ, séparés par des séparateurs de champs dans des enregistrements de longueur variable.
· champ (dans une clé) : une clé est attachée à chaque champ. Elle définit quelles données seront prises en compte lors de l'opération de tri. Le champ d'une clé peut être défini comme plusieurs champs dans un enregistrement.
· clé : lors du tri des données, une clé est utilisée pour définir les données à trier et l'ordre de tri à utiliser. Une clé peut prendre la forme de données ASCII ou numérique, et peut être triée dans un ordre ascendant ou descendant. S'il existe plusieurs clés, elles sont appliquées dans l'ordre, la première clé étant celle dont le tri est plus pertinent.
· trier : l'action de classer des enregistrements en fonction d'une liste de clés. Si deux enregistrements sont identiques, une clé moins pertinente est utilisée.
· doublon : si deux enregistrements ont des clés identiques, ils sont considérés comme des doublons.
· dernier recours : même si deux enregistrements ont des clés identiques, ils doivent quand même être triés dans le fichier de sortie. Le dernier recours est une clé additionnelle ne modifiant pas le statut de "doublon" de l'enregistrement. Le dernier recours peut être un tri stable (le premier enregistrement rencontré dans les données d'entrée sera la premier dans le fichier de sortie) ou prend la ligne entière comme clé de tri.
