tHDFSPut

Avertissement

Ce composant est disponible dans la Palette de votre studio si vous avez souscrit à l'édition correspondante de Talend Big Data Studio.

Propriétés du tHDFSPut

Famille de composant

Big Data/Hadoop

 

Fonction

tHDFSPut charge les données dans le système de fichiers Hadoop.

Objectif

tHDFSPut se connecte au système de fichiers distribués Hadoop pour charger de gros fichiers avec une performance optimisée.

Basic settings

Use an existing connection

Cochez cette case et sélectionnez le composant tHDFSConnection adéquat à partir de la liste Component pour réutiliser les paramètres d’une connexion que vous avez déjà définie.

Note

Lorsqu’un Job contient un Job parent et un Job enfant, la liste Component list présente uniquement les composants de connexion du Job du même niveau. Si vous souhaitez utiliser une connexion existant dans un autre niveau, vous pouvez utiliser les paramètres dynamiques Dynamic settings afin de partager la connexion souhaitée. Dans ce cas, assurez-vous que le nom de la connexion est unique et distinct à travers les deux niveaux de Jobs. Pour plus d'informations concernant les paramètres dynamiques Dynamic settings, consultez le Guide utilisateur de Talend Data Integration Studio.

 Distribution

Sélectionnez dans la liste déroulante le produit que vous utilisez comme distribution d'Hadoop.

Les options de la liste varient selon le composant que vous utilisez.

 Hadoop version

Sélectionnez la version de la distribution d'Hadoop que vous utilisez.

 

NameNode URI

Saisissez l'URI du NameNode Hadoop.

 

User name

Nom d'utilisateur de HDFS.

 

Group

Identifiant de l’utilisateur et nom du groupe sous lesquels les instances HDFS ont été lancées. Ce champ peut être disponible ou indisponible selon la distribution que vous utilisez.

 

Use kerberos authentication

Si vous accédez au cluster Hadoop fonctionnant avec la sécurité de Kerberos, cochez cette case, puis saisissez le "principal name" de Kerberos pour le NameNode dans le champ affiché.

Cette case est disponible ou indisponible selon la distribution d'Hadoop à laquelle vous vous connectez.

 

HDFS directory

Parcourez votre système ou saisissez le chemin d'accès au répertoire de HDFS dans lequel sont stockées les données à utiliser.

 

Local directory

Dossier local où sont stockés les fichiers obtenus de HDFS.

 

Overwrite file

Option permettant d’écraser ou non les fichiers avec le nouveau fichier.

 

Files

Dans la zone Files, les champs à remplir sont :

- File mask : saisissez le nom du fichier qui sera sélectionné dans HDFS. Vous pouvez utiliser des expressions régulières.

- New name: renommez le fichier obtenu.

 

Die on error

La case est cochée par défaut. Décochez la case afin d’ignorer la ligne d’erreur et finir le processus de lignes sans erreur.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Utilisation

Le composant combine la connexion HDFS et l’extraction de données, de cette façon il est souvent utilisé comme simple composant sous-job mais peut également être utilisé comme sortie ou comme composant final d’un Job.

Limitations

La version 1.6+ de JRE est requise pour exécuter le composant.

Scénario associé

Pour un scénario associé, consultez la section intitulée « Scénario : Traiter des données avec le système de fichiers distribués Hadoop ».
