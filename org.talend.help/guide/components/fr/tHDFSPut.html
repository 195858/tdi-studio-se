tHDFSPut

Ce composant est disponible dans la Palette de votre studio si vous avez souscrit à l'édition correspondante de Talend Enterprise Data Integration Big Data edition.

Propriétés du tHDFSPut

Famille de composant

Big Data/Hadoop

 

Fonction

tHDFSPut charge les données dans le système de fichiers Hadoop.

Objectif

tHDFSPut se connecte au système de fichiers distribués Hadoop pour charger de gros fichiers avec une performance optimisée.

Basic settingsHadoop version

Sélectionnez une version de Hadoop dans la liste :

Hortonworks Data Platform 1.0 : Vous devez spécifier l'accès à Hortonworks Data Platform 1.0 supportant les projets Apache Hadoop, notamment HDFS, MapR, Hive, HBase, Pig et Zookeeper.

Apache 1.0.0 : Vous devez spécifier l'accès à Apache Hadoop 1.0.0 à utiliser.

Apache 0.20.204 : Vous devez spécifier l'accès à Apache 0.20.204 à utiliser.

Cloudera 0.20 CDH3UI : Vous devez spécifier l'accès à Cloudera 0.20 Hadoop (CDH3 Update1) à utiliser.

Apache 0.20.2 : Vous devez spécifier l'accès à Apache 0.20.2 à utiliser.

MapR : Vous devez spécifier l'accès à la distribution MapR pour Apache Hadoop à utiliser.

 HostAdresse IP du serveur du système de fichiers distribués Hadoop.

 

Port

Numéro du port d’écoute du serveur HDFS.

 

User name

Identifiant de l’utilisateur HDFS.

 

Group

Identifiant de l’utilisateur et nom du groupe sous lesquels les instances HDFS ont été lancées. Ce champ est disponible uniquement lorsque vous sélectionnez Apache 0.20.2 ou MapR dans la liste Hadoop version.

 

HDFS directory

Emplacement des fichiers source dans HDFS.

 

Local directory

Dossier local où sont stockés les fichiers obtenus de HDFS.

 

Overwrite file

Option permettant d’écraser ou non les fichiers avec le nouveau fichier.

 

Files

Dans la zone Files, les champs à remplir sont :

- File mask : saisissez le nom du fichier qui sera sélectionné dans HDFS. Vous pouvez utiliser des expressions régulières.

- New name: renommez le fichier obtenu.

 

Die on error

La case est cochée par défaut. Décochez la case afin d’ignorer la ligne d’erreur et finir le processus de lignes sans erreur.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Utilisation

Le composant combine la connexion HDFS et l’extraction de données, de cette façon il est souvent utilisé comme simple composant sous-job mais peut également être utilisé comme sortie ou comme composant final d’un Job.

Limitations

tHDFSGet et tHDFSPut doivent être exécutés à partir de la version JRE 1.6+.

Scénario associé

Pour un scénario associé, consultez la section intitulée « Scénario : Traiter des données avec le système de fichiers distribués Hadoop ».
