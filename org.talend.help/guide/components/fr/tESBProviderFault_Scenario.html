Scénario : Retourner un message Fault

Ce scénario Java décrit comment utiliser un composant tESBProviderFault afin de retourner un message Fault.

Vous devez créer deux Jobs pour ce scénario :

    • un Job fournisseur
    • un Job consommateur

Créer un Job fournisseur

Cette section vous montre comment créer un Job fournisseur.

      • Déposez les composants suivants de la Palette dans l'espace de modélisation graphique : un tESBProviderRequest, trois tLogRow, deux tJavaFlex et un tESBProviderFault.
      • Connectez les composants comme dans la capture d'écran.
      • Configurez les propriétés du composant tESBProvider Request_1.
      • Modifiez le schéma du composant tESBProvider Request_1.
      • Configurez les propriétés du composant tLogRow_1.
      • Modifiez le schéma du composant tLogRow_1 component.
      • Configurez les propriétés du composant tJavaFlex_1. Ajoutez le code suivant dans le champ Main code, comme ci-dessous.
        Document requestTalendDoc = row2.payload;
        org.dom4j.Document requestDoc = requestTaldndDoc.getDocument();
        org.dom4j.Element rootElement = requestDoc.getRootElement();
        String text = rootElement.getTextTrim();
        System.out.println("### " + text);
        row3.content = text;
      • Configurez le schéma du composant tJavaFlex_1.
      • Configurez les propriétés du composant tLogRow_2.
      • Configurez le schéma du composant tLogRow_2.
      • Définissez les propriétés du composant tJavaFlex_2. Ajoutez le code suivant dans le champ Main code comme suit.
        String fMessage = row4.content;
        String faultText = "Fault message text: " + fMessage + "!";

        org.dom4j.Document faultDoc = org.dom4j.DocumentHelper.createDocument();
        faultDoc.addElement(new org.dom4j.QName("response",

        org.dom4j.Namespace.get("http://talend.org/esb/service/job"))
        ).addText(faultText);

        Document faultTalendDoc = new Document();
        faultTalendDoc.setDocument(faultDoc);
        row5.payload = faultTalendDoc;
      • Configurez le schéma du composant tJavaFlex_2.
      • Configurez les propriétés du composant tLogRow_3.
      • Configurez le schéma du composant tLogRow_3.
      • Définissez les propriétés du composant tESBProviderFault_1.
      • Configurez le schéma du composant tESBProviderFault_1.
      • Le Job peut être exécuté sans erreur.

Créer un Job consommateur

Cette section explique comment créer un Job consommateur.

      • Déposez les composants suivants de la Palette dans l'espace de modélisation graphique : un tESBConsumer, deux tLogRow, un JavaFlex et un tFixedFlowInput.
      • Connectez les composants comme dans la capture d'écran.
      • Configurez le schéma du composant tFixedFlowInput_1.
      • Configurez les propriétés du composant tFixedFlowInput_1.
      • Définissez les propriétés du composant tJavaFlex_1. Ajoutez le code suivant dans le champ Main code.
        org.dom4j.Document doc = org.dom4j.DocumentHelper.parseText(row1.payloadString);
        Document talendDoc = new Document();
        talendDoc.setDocument(doc);
        row2.payload = talendDoc;
      • Configurez le schéma du composant tJavaFlex_1.
      • Exécutez le Job fournisseur. Dans le log d'exécution, vous pouvez voir :
        ...
        web service [endpoint: http://127.0.0.1:8008/esb/provider] published
        ...
      • Ouvrez Service Configuration et saisissez le chemin d'accès au WSDL : http://127.0.0.1.8088/esbprovider?WSDL
      • Cliquez sur le bouton afin de récupérer le nom du port et le nom de l'opération.
      • Configurez le schéma d'entrée, le schéma de réponse (response) et le schéma d'erreur (fault) du composant tESBComsumer_1.
      • Arrêtez le Job fournisseur.
      • Configurez les propriétés du composant tLogRow_1.
      • Configurez le schéma du composant tLogRow_1.
      • Définissez les propriétés du composant tLogRow_2.
      • Configurez le schéma du composant tLogRow_2.
      • Le Job peut être exécuté sans erreur.
