Avertissement

Ce composant est disponible dans la Palette du Studio uniquement si vous avez souscrit à l'édition correspondante de Data Quality Service Hub Studio.

Propriétés du tUniservRTConvertName

Famille de composant

Data Quality

 

Fonction

tUniservRTConvertName analyse un nom complet par rapport au contexte. S'il s'agit de personnes, il divise le nom en segments (nom, prénom, civilité, préfixes, suffixes etc.) et créer une clé d'adresse.

Le composant reconnaît les adresses d'entreprise ou d'institution et est capable de fournir la fiche de l'organisme séparément. Il divise également les lignes contenant des informations sur plusieurs personnes. De plus, il reconnaît certains modèles n'appartenant pas aux informations de nom, dans la ligne de nom (numéro client, notes, etc.) et les supprime ou les déplace dans certains champs mémo spécifiques.

Objectif

Le tUniservRTConvertName fournit une base pour une structure et une population uniformes de noms de personnes et d'entreprises dans la base de données, ainsi que des titres personnalisés.

Basic settings

Schema et Edit schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (Built-in) soit distant dans le Repository.

 

Host name

Nom de l'hôte du serveur, entre guillemets doubles.

 

Port

Numéro du port du serveur, entre guillemets doubles.

 

Service

Le nom/type du service est "cname_d" par défaut. Saisissez un nouveau nom si nécessaire (si le suffixe du service change, par exemple), entre guillemets doubles. Les services disponibles sont :

Allemagne "cname_d"

Italie "cname_i"

Autriche "cname_a"

Pays-Bas "cname_nl"

Suisse "cname_ch"

Belgique "cname_b"

France "cname_f"

Espagne "cname_e"

 

Use rejects

Cochez cette case pour écrire séparément en sortie les ensembles de données séparément d'une certaine classe de résultats lors de l'analyse des noms. Saisissez la classe respective des résultats dans le champ if result class is greater or equal to.

Si cette case n'est pas cochée, les ensembles sont tout de même écrits en sortie via la connexion Main, même si l'analyse a échoué.

Si cette case est cochée mais que la connexion Rejects n'a pas été établie, les ensembles sont simplement triés quand l'analyse échoue.

Scénario : Analyse d'une ligne de nom et attribution d'un titre

Ce scénario décrit un Job vous permettant d'analyser le nom de personnes dans un fichier et de leur attribuer un titre.

Le fichier d'entrée de ce scénario est déjà sauvegardé dans le Repository, toutes les métadonnées du schéma sont donc disponibles.

Note

Veuillez noter que les données d'une source d'entrée doivent toutes être liées au même pays.

  1. Dans la vue Repository, déroulez le nœud Metadata et le répertoire dans lequel est sauvegardé le fichier. Glissez ce fichier dans l'espace de modélisation graphique.

    La boîte de dialogue suivante s'ouvre :

  2. Sélectionnez le composant tFileInputDelimited et cliquez sur OK pour fermer la boîte de dialogue.

    Le composant s'affiche dans l'espace de modélisation graphique. Le fichier utilisé dans ce scénario est appelé SampleAddresses..

  3. Glissez les composants suivants de la Palette à l'espacé de modélisation graphique : deux composants tMap, un tUniservRTConvertName, et untFileOutputDelimited.

  4. Reliez les composants à l'aide de liens Row > Main.

    Acceptez le schéma du tUniservRTConvertName en cliquant sur Yes dans la fenêtre qui apparaît.

  5. Double-cliquez sur le tMap_1 pour ouvrir la fenêtre de mapping. À gauche se trouve la structure de la source d'entrée et à droite se trouve le schéma du tUniservRTConvertName (et donc l'entrée du Job). En bas se trouve le Schema editor, où vous pouvez trouver et éditer les attributs de chaque colonne.

  6. Assignez les colonnes de la source d'entrée à leurs colonnes respectives du tUnivservRTConvertName. Pour ce faire, sélectionnez une colonne de la source d'entrée et glissez-la dans la colonne du même nom du côté droit. Si les champs du fichier d'entrée doivent être passés au fichier de sortie, comme les adresses ou les IDs, vous devrez définir des champs supplémentaires.

  7. Cliquez sur OK pour fermer la boîte de dialogue.

  8. Double-cliquez sur le tUniservRTConvertName pour ouvrir sa vue Basic settings.

  9. Renseignez les informations concernant le serveur et spécifiez le pays lié au service.

  10. Double-cliquez sur le tMap_3 pour ouvrir la fenêtre de mapping. À gauche se trouve le schéma du tUniservRTConvertName (et donc la sortie du Job) et à droite le schéma du fichier de sortie.

  11. Cliquez sur OK pour fermer la fenêtre.

  12. Double-cliquez sur le tFileOutputDelimited et saisissez les détails du fichier de sortie.
