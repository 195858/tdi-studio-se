Scénario 1 : Lire une table SAP

Les composants SAP (tSAPInput et tSAPOutput) ainsi que l'assistant SAP sont basés sur des bibliothèques validées et fournies par SAP (JCO) permettant à l'utilisateur d'appeler les fonctions RFC ou BAPI et de récupérer les données du système SAP au niveau des Tables, RFC ou BAPI.

Ce scénario utilise l'assistant SAP guidant l'utilisateur à travers les différentes étapes de création de connexion SAP et des fonctions RFC et BAPI. L'assistant SAP est uniquement disponible pour les utilisateurs Talend Integration Suite. Si vous êtes utilisateur de Talend Open Studio, vous devez configurer les propriétés du composant tSAPInput manuellement.

Ce scénario Java utilise l'assistant SAP pour créer un connexion au système SAP et pour appeler une fonction RFC permettant de lire directement dans une table appelée SFLIGHT du système SAP. Puis il affiche dans Talend Open Studio la structure de la table SFLIGHT stockée dans le système SAP.

    • Créez une connexion au système SAP à l'aide de l'assistant SAP. Dans ce scénario, la connexion SAP est appelée SAPdemo.Talend Open Studio
    • Appelez la fonction RFC appelée RFC_READ_TABLE à l'aide de l'assistant SAP pour accéder à la table du système SAP et pour visualiser sa structure.Talend Open Studio
    • Dans le champ Name filter, saisissez RFC* et cliquez sur le bouton Search pour afficher toutes les fonctions RFC disponibles.
    • Sélectionnez RFC_READ_TABLE pour afficher la schéma décrivant la structure de cette table.

La vue à droite de l'assistant affiche les métadonnées de la fonction RFC_READ_TABLE et permet de définir les paramètres nécessaires.

L'onglet Document affiche le document html SAP de la fonction RFC_READ_TABLE.

L'onglet Parameter fournit les informations sur les paramètres requis par la fonction RFC_READ_TABLE pour pouvoir retourner les valeurs des paramètres.

    • Dans la vue Parameter, cliquez sur l'onglet Input pour lister les paramètres d'entrée. Dans ce scénario, les paramètres d'entrée requis par la fonction RFC_READ_TABLE sont DELIMITER, QUERY_TABLE et ROWCOUNT. Vous pouvez supprimer les paramètres d'entrée superflus dans la vue Test it.
    • Dans la vue Parameter, cliquez sur l'onglet Output pour lister les paramètres qui seront retournés par le fonction RFC_READ_TABLE. Dans ce scénario, les paramètres sont FILENAME, OFFSET, LENGTH, TYPE et FIELDTEXT.
    • Dans la vue Parameter, cliquez sur l'onglet Table pour afficher la structure de la table SFLIGHT.

La vue Test it vous permet d'ajouter ou de supprimer des paramètres d'entrée en fonction de la fonction appelée. Dans ce scénario, nous voulons récupérer la structure de la table SFLIGHT et non les données.

    • Placez le curseur de la souris sur dans le champ Value et cliquez sur Launch en bas de la vue pour afficher les valeurs de paramètres retournées par la fonction RFC_READ_TABLE. Dans ce scénario, le délimiteur est "|", la table à lire est SFLIGHT et le numéro de ligne est 20.

Ce nouveau schéma RFC_READ_TABLE est affiché dans le dossier SAPdemo du Repository.

Pour récupérer la structure de la table et l'afficher dans Talend Open Studio :

    • Dans le Repository, cliquez-droit sur RFC_READ_TABLE dans le dossier SAPdemo, sélectionnez tSAPInput dans la liste de composants et déposez-le dans l'espace de modélisation.
    • Cliquez et déposez les composants tFilterColumns et tLogRow dans l'espace de modélisation.
    • Connectez les trois composants à l'aide de liens de type Row Main.
    • Dans l'espace de modélisation, sélectionnez tSAPInput et cliquez sur la vue Component pour afficher ses propriétés.

Les propriétés du composant tSAPInput apparaissent automatiquement puisque le schéma est stocké dans le dossier Metadata et que le composant est initialisé par l'assistant SAP. Dans la zone Initialize input, nous pouvons voir les trois paramètres d'entrée nécessaires pour la fonction RFC_READ_TABLE.

Dans la zone Extract data, nous pouvons voir les deux colonnes superflues : WA et TEXT puisque nous souhaitons uniquement voir la structure de la table SFLIGHT et non les données.

    • En bas de la vue Basic settings du composant tSAPInput et dans la liste Type (for iterate), sélectionnez table_output et saisissez "FIELDS" dans le champ TableName or StructureName.

Pour filtrer le schéma de la table SFLIGHT, nous utilisons le composant tFilterColumns :

    • Dans l'espace de modélisation, sélectionnez tFilterColumns et cliquez sur la vue Component pour paramétrer ses propriétés.
    • Cliquez sur Edit schema pour afficher l'éditeur du tFilterColumns.
    • Configurez le schéma de sortie en fonction du libellé des colonnes du schéma d'entrée. Pour ce scénario, nous souhaitons supprimer les colonnes WA et TEXT pour obtenir un schéma de sortie contenant les colonnes suivantes : FIELDNAME, OFFSET, LENGTH, TYPE et FIELDTEXT.
    • Cliquez sur OK pour valider les changements.
    • Dans l'espace de modélisation, sélectionnez le composant tLogRow et cliquez sur la vue Component pour paramétrer ses propriétés. Pour plus d'informations, see tLogRow..
    • Enregistrez votre job et appuyez sur F6 pour l'exécuter.

Le composant tSAPInput récupère les cinq colonnes de la table SFLIGHT à partir du système SAP et le composant tLogRow les affiche dans la console.

Scénario 2 : Récupérer les métadonnées d'un système SAP

Ce scénario utilise l'assistant SAP guidant l'utilisateur à travers les différentes étapes de création de connexion SAP et des fonctions RFC et BAPI. L'assistant SAP est uniquement disponible pour les utilisateurs Talend Integration Suite. Si vous êtes utilisateur de Talend Open Studio, vous devez configurer les propriétés du composant tSAPInput manuellement.

Ce scénario Java utilise un assistant SAP pour créer une connexion au système SAP, appelle une fonction BAPI pour récupérer les informations d'une entreprise stockées dans le système SAP et les affiche dans Talend Open Studio.

Les figures ci-dessous affiche les paramètres du système SAP permettant d'accéder aux informations sur l'entreprise que nous souhaitons lire dans Talend Open Studio à l'aide du composant tSAPInput.

    • Créez une connexion au système SAP à l'aide de l'assistant SAO. Dans ce scénario, la connexion SAP est appelée SAPdmo.Talend Open Studio
    • Appelez la fonction BAPI appelée BAPI_COMPANY_GETDETAIL à l'aide de l'assistant SAP pour accéder au document HTML BAPI stocké dans le système SAP et consultez les informations sur l'entreprise.Talend Open Studio
    • Dans le champ Name filter, saisissez BAPI* et cliquez sur le bouton Search pour afficher toutes les fonctions BAPI disponibles.
    • Sélectionnez BAPI_COMPANY_GETDETAIL pour afficher le schéma décrivant les informations de l'entreprise.

La vue à droite de l'assistant affiche les métadonnées de la fonction BAPI_COMPANY_GETDETAIL et vous permet de configurer les paramètres nécessaires.

La vue Document affiche le document HTML SAP de la fonction BAPI_COMPANY_GETDETAIL.

La vue Parameter fournit les informations sur les paramètres d'entrée et de sortie nécessaire à la fonction BAPI_COMPANY_GETDETAIL pour retourner les valeurs.

    • Dans la vue Parameter, cliquez sur l'onglet Input pour lister le ou les paramètre(s) d'entrée. Dans ce scénario, il n'y a qu'un seul paramète d'entrée requis par la fonction BAPI_COMPANY_GETDETAIL. Ce paramètre s'appelle COMPANYID.
    • Dans la vue Parameter, cliquez sur l'onglet Output pour lister les paramètres de sortie retournés par la fonction BAPI_COMPANY_GETDETAIL. Dans ce scénario, il y a deux paramètres de sortie : COMPANY_DETAIL et RETURN.

Chacun des paramètres de "structure" est composé de nombreux paramètres "single".

La vue Test it permet d'ajouter ou de supprimer des paramètres d'entrée en fonction de la fonction appelée. Dans ce scénario, nous souhaitons récupérer les métadonnées du paramètre "structure" appelé COMPANY_DETAIL qui est composé de 14 paramètres "single".

    • Placez le curseur de la souris sur dans le champ Value et cliquez sur Launch en bas de la vue pour afficher la valeur de chaque paramètre "single" retourné par la fonction BAPI_COMPANY_GETDETAIL.
    • Cliquez sur Next pour afficher la liste des métadonnées pour la fonction BAPI_COMPANY_GETDETAIL.
    • Cliquez sur Finish pour fermer l'assistant.

Le nouveau schéma BAI_COMPANY_GETDETAIL apparaît dans le dossier SAPdemo du Repository.

Pour récupérer et afficher les métadonnées dans Talend Open Studio :

    • Dans le Repository, cliquez-droit sur BAPI_COMPANY_GETDETAIL dans le dossier SAPdemo, sélectionnez le composant tSAPInput dans la liste des composants et déposez-le dans l'espace de modélisation.
    • Cliquez et déposez les composants tFilterColumns et tLogRow dans l'espace de modélisation.
    • Connectez les trois composants à l'aide de liens Row Main.
    • Dans l'espace de modélisation, sélectionnez le composant tSAPInput et cliquez sur la vue Component pour afficher ses propriétés.

Les propriétés du composant tSAPInput apparaissent automatiquement puisque le schéma est stocké dans le dossier Metadata et que le composant est initialisé par l'assistant SAP. Dans la zone Initialize input, nous pouvons voir les paramètres d'entrée nécessaires à la fonction BAPI_COMPANY_GETDETAIL.

Dans la zone Extract data, nous pouvons voir tous les paramètres "single" de sortie. Dans ce scénario, nous souhaitons lire dans Talend Open Studio les métadonnées des 14 premiers paramètres "single" du paramètre "structure" COMPANY_DETAIL. Pour cela, nous devons donc supprimer tous les paramètres de type "single" du paramètre RETURN de type "structure".

    • Dans la liste Type, sélectionnez output_structure et dans TableName or StructureName saisissez "COMPANY_DEAIL".
    • Dans l'espace de modélisation, sélectionnez tFilterColumns et cliquez sur la vue Component pour paramétrer ses propriétés.
    • Cliquez sur Edit schema pour afficher l'éditeur du tFilterColumns.
    • Configurez le schéma de sortie en fonction du libellé des colonnes du schéma d'entrée. Dans ce scénario, nous souhaitons supprimer toutes les colonnes après LANG_ISO.
    • Cliquez sur OK pour valider les changements.
    • Dans l'espace de modélisation, sélectionnez le composant tLogRow et cliquez sur la vue Component pour paramétrer ses propriétés. Pour plus d'informations, see tLogRow..
    • Enregistrez le job et appuyez sur F6 pour l'exécuter.

Le composant tSAPInput récupère du système SAP les métadonnées du paramètre COMPANY_DETAIL de type "structure" et le composant tLogRow affiche les informations dans la console.
