tHCatalogOutput

Ce composant est disponible dans la Palette du Studio uniquement si vous avez souscrit à l'édition correspondante de Talend Enterprise Data Integration Big Data edition.

Propriétés du tHCatalogOutput

Famille de composant

BigData/HCatalog

 

Fonction

Ce composant vous permet d'écrire des données dans une table gérée par HCatalog à l'aide d'un flux Talend.

Objectif

Le composant tHCatalogOutput écrit des données dans une table gérée par HCatalog à l'aide d'un flux Talend.

Basic settings

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository. Dans ce composant, le schéma est en lecture seule.

 

 

Built-in : Le schéma est créé et conservé ponctuellement pour ce composant seulement. Voir également le Guide utilisateur de Talend Enterprise Studio.

 

 

Repository : Le schéma existe déjà et il est stocké dans le Repository. Ainsi, il peut être réutilisé. Voir également le Guide utilisateur de Talend Enterprise Studio.

 

HCatalog version

Sélectionnez une version de Hadoop dans la liste :

Hortonworks Data Platform 1.0 : Vous devez spécifier l'accès à Hortonworks Data Platform 1.0 supportant les projets Apache Hadoop, notamment HDFS, MapR, Hive, HBase, Pig et Zookeeper.

 

NameNode URI

Saisissez l'URI du NameNode dans HDFS.

 

File name

Parcourez votre système ou saisissez le chemin d'accès au fichier dans lequel vous souhaitez écrire les données. Ce fichier est automatiquement créé s'il n'existe pas.

 

Action

Sélectionnez une opération sur la base de données dans HDFS :

Create : Créer un fichier avec des données à l'aide du nom de fichier défini dans le champ File Name.

Overwrite : Ecraser les données dans le fichier spécifié dans le champ File Name.

Append : Insérer les données dans le fichier spécifié dans le champ File Name. Le fichier spécifié est automatiquement créé s'il n'existe pas.

 

Templeton hostname

Renseignez ce champ avec l'URL du service Web Templeton.

Templeton est une API de services Web pour Hadoop. Il vous permet de déplacer les données directement dans /en dehors de HDFS via WebHDFS. Pour plus d'informations concernant Templeton, consultez Templeton 0.1.0 (en anglais).

 

Templeton port

Renseignez ce champ avec le port de l'URL du service Web Templeton. Par défaut, cette valeur est 50111.

Templeton est une API de services Web pour Hadoop. Il vous permet de déplacer les données directement dans /en dehors de HDFS via WebHDFS. Pour plus d'informations concernant Templeton, consultez Templeton 0.1.0 (en anglais).

 

Database

Renseignez ce champ pour spécifier une base de données existant dans HDFS.

 

Table

Renseignez ce champ pour spécifier une table existant dans HDFS.

 

Partition

Renseignez ce champ pour spécifier une ou plusieurs partition(s) pour l'opération de partition sur la table spécifiée. Lorsque vous spécifiez différentes partitions, utilisez une virgule pour séparer chaque groupe de deux partitions. Utilisez également des guillemets doubles afin d'entourer la chaîne de caractères de la partition.

Pour plus d'informations concernant les partitions, consultez Operation on Partitions in Hive (en anglais).

  Username

Renseignez ce champ avec le nom d'utilisateur de connexion à la base de données.

  File location

Renseignez ce champ avec le chemin d'accès à l'emplacement où est stocké le fichier des données source.

 

Die on error

Cette case est décochée par défaut, afin d'ignorer les lignes en erreur et de terminer le traitement avec les lignes sans erreur.

Advanced settings Retrieve the HCatalog logs Cochez cette case pour récupérer les fichiers de log générés durant les opérations de HCatalog.
  Standard Output Folder

Renseignez ce champ avec le chemin d'accès où sont stockés les fichiers de log.

Ce champ est activé lorsque vous cochez la case Retrieve the HCatalog logs.

 

Error Output Folder

Renseignez ce champ avec le chemin d'accès où sont stockés les fichiers de log d'erreurs.

Ce champ est activé lorsque vous cochez la case Retrieve the HCatalog logs.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du Job ainsi qu'au niveau de chaque composant.

Utilisation

Ce composant de sortie est généralement utilisé avec un composant d'entrée.

Limitation

La connaissance de Hive Data Definition Language et de HCatalog Data Definition Language est nécessaire. Pour plus d'informations concernant Hive Data Definition Language, consultez Hive Data Definition Language (en anglais). Pour plus d'informations concernant HCatalog Data Definition Language, consultez HCatalog Table Management (en anglais).
