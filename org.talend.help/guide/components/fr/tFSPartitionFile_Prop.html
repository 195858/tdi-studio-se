Propriétés du tFSPartitionFile

Famille de composant
FileScale
Fonction
Le composant tFSPartitionFile vous permet de partitionner des données en masse d'un fichier d'entrée avec les méthodes de partitionnement Hash ou Round-Robin avant de les écrire dans un fichier de sortie. Cela permet de gérer de très grandes tables.
Ce composant possède des fonctionnalités de partitionnement de fichiers à gros volume en temps réel. Afin d'optimiser les performances, le composant trie généralement les données avant de les traiter.
Objectif
Permet de partitionner des données en masse avant de les écrire dans un fichier de sortie.
Basic settings
Schema et Edit Schema
Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.
Cliquez sur Edit Schema pour modifier le schéma.

Si vous modifiez le schéma, il se met automatiquement en built-in.

 
 
Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et job designs. Voir également : Paramétrer un schéma du Repository du Guide Utilisateur de Talend Integration Suite Studio.
 
 
Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également : Paramétrer un schéma built-in du Guide Utilisateur de Talend Integration Suite Studio.
 
Property type
Peut être Built-in ou Repository.
 
 
Built-in : Propriétés utilisées ponctuellement.
 
 
Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l'aide des données collectées.
 
Input File Name
Nom du fichier contenant les données que vous souhaitez partitionner.
 
Output File Name
Nom du fichier dans lequel vous souhaitez écrire les données partitionnées.

Les fichiers de sortie générés se verront ajouter un postfixe avec un numéro auto-incrémenté qui dépend du nombre de partitions défini.

 
Record separator (char)
Caractère, chaîne de caractères ou expression régulière pour séparer les enregistrements (lignes).
 
Field separator (char)
Caractère, chaîne de caractères ou expression régulière pour séparer les champs dans un enregistrement.
 
Header
Nombre d'enregistrements à ignorer au début du fichier.
 
Footer
Nombre d'enregistrements à ignorer à la fin du fichier.
 
Number of partitions
Nombre de partitions dans le fichier.
 
Partition
Sélectionnez dans la liste la méthode de partition que vous souhaitez utiliser :
Round-robin : Les enregistrements sont partitionnés via la méthode Round-Robin, afin que chaque partition contienne plus ou moins le même nombre de lignes et que la répartition de charge soit effectuée. Puisqu'il n'y a pas de clé de partition, les lignes sont distribuées au hasard dans toutes les partitions.
Hash : Les enregistrements sont hachés en partitions basées sur la valeur d'une colonne clé ou de colonnes clé sélectionnée(s) dans le schéma du fichier.
Cochez la case Partition Key correspondant à la colonne (aux colonnes) que vous souhaitez utiliser comme base de partitionnement des données.

Utilisation
Ce composant gère des fichiers et ne nécessite donc pas de flux d'entrée ou de sortie. Il est utilisé pour effectuer une jointure sur des données to dans des fichiers à gros volume.
Limitation
La limitation dépend des limites imposées par la mémoire physique et par l'architecture du processeur. Par exemple, la longueur totale des fichiers traités ne peut excéder la limite système pour les fichiers à gros volume (valeur maximum de 64 bits signés).
