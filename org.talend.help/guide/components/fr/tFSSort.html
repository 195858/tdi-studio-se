tFSSort

Avertissement

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition de Talend Big Data Studio. Pour plus d’informations concernant le mode Massively Parallel eXtension, consultez le Guide utilisateur de Talend Big Data Studio.

Propriétés du tFSSort

Famille de composant

FileScale

 

Fonction

Le composant tFSSort trie des données d’entrée sur certains critères et dans un certain ordre en fonction d’une ou plusieurs colonnes. Ce composant permet de trier très rapidement les fichiers volumineux.

Objectif

Le tFSSort d’établir des métriques et des tables de classification.

Basic settings

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma.

Note

Si vous modifiez le schéma, il se met automatiquement en built-in.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et job designs. Voir également le Guide utilisateur de Talend Data Integration Studio.

 

 

Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également le Guide utilisateur de Talend Data Integration Studio.

 

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Input File Name

Nom du fichier contenant les données que vous souhaitez filtrer.

 

Output File Name

Nom du fichier dans lequel vous souhaitez écrire les données filtrées.

 

Record separator (char)

Caractère, chaîne ou expression régulière séparant les lignes.

 

Field separator (char)

Caractère, chaîne ou expression régulière séparant les champs.

 

Header

Nombre de lignes d’en-tête à ignorer en début de fichier.

 

Footer

Nombre de lignes à ignorer en fin de fichier.

 

Criteria

Cliquez sur le bouton plus [+] pour ajouter autant de lignes que nécessaire pour vérifier que le fichier source est bien trié.

Schema column : Cliquez dans la cellule est sélectionné le nom de la colonne sur laquelle vous souhaitez vous baser pour trier le fichier

Note

L’ordre est essentiel puisqu’il détermine la priorité de tri.

Sort type : Cliquez dans le cellule et sélectionnez le type de tri : numérique ou alphabétique.

Order type : Cliquez dans la cellule et sélectionnez l’ordre de tri : ascendant ou descendant.

Advanced settings

Generate FSLang File

Cochez cette case pour générer le fichier FSLang correspondant à votre Job et cliquez sur le bouton [...] à côté du champ FSLang File Name pour spécifier son chemin d’accès et son nom.

 

Assign FileScale Path

Cochez cette case puis cliquez sur le bouton [...] à côté du champ FileScale Path pour sélectionner le fichier exécutable correspondant au programme FileScale permettant d’exécuter le composant.

 

Specify Number of Process Child

Cochez cette case et saisissez le nombre de processus fils à utiliser pour l’exécution de l’agrégation.

 

Custom FileScale Parameter (separated by ,)

Saisissez les paramètres pour chaque opération spécifique que vous souhaitez ajouter à l’appel de l’exécutable FileScale.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Utilisation

Ce composant traite des fichiers, ainsi il ne nécessite pas de flux de données en entrée ou en sortie. Il sert à vérifier les données contenues dans des fichiers volumineux.

Limitation

La limitation dépend des limites de la mémoire physique et de l’architecture du processeur. Par exemple, la taille totale des fichiers traités ne peut pas excéder la limite du système de fichiers pour supporter les fichiers très volumineux (maximum de la valeur 64 bits signée).

Scénario : Trier les entrées d’un fichier volumineux

Ce scénario a été créé en Java et décrit un Job triant, en quelques minutes, un très gros volume de données (10 000 000 lignes) d’un fichier volumineux en fonction de deux critères.

Avertissement

Assurez-vous d’avoir bien enregistré localement le fichier exécutable correspondant à votre système d’exploitation afin de pouvoir indiquer son chemin d’accès dans Talend Data Integration Studio.

Dans ce scénario, vous avez déjà centralisé le schéma du fichier d’entrée dans le Repository. Pour plus d’informations concernant la création et l’utilisation de métadonnées de schémas dans le Repository, consultez le Guide utilisateur de Talend Data Integration Studio.

Le fichier d’entrée contient 10 colonnes : id, surname, firstname, zipcode, city, dateofbirth, streetname, streetnr, statecode, et state.

  • Dans le Repository, développez le nœud Metadata et le nœud de la métadonnée que vous venez de créer contenant les schémas d’entrée et glissez ce nœud dans le Job designer.

    La boîte de dialogue [Components] apparaît.

  • Sélectionnez le composant tFSSort dans la liste et cliquez sur OK pour fermer la boîte de dialogue.

    Le composant tFSSort apparaît dans le Job designer.

  • Double-cliquez sur le composant tFSSort pour paramétrer ses propriétés dans l’onglet Basic settings de la vue Component.

Tous les champs de propriétés du tFSSort sont automatiquement renseignés. Si vous n’avez pas défini le schéma d’entrée dans les métadonnées du Repository, renseignez manuellement ces informations après avoir sélectionné le mode Built-in dans les listes Schema Type et Property Type.

  • Dans le champ Output File Name, indiquez le nom et le chemin d’accès au fichier de sortie contenant les données triées.

  • Dans la zone Criteria, cliquez sur le bouton [+] pour ajouter des colonnes à la liste, puis sélectionnez les colonnes du schéma sur lesquels vous souhaitez baser votre tri.

    Dans ce scénario, l’objectif est de trier les données en fonction des colonnes city et surname et de trier les données par ordre alphabétique ascendant.

  • Dans les colonnes Sort type, cliquez dans la cellule et sélectionnez alpha pour indiquer que vous souhaitez trier les données par ordre alphabétique.

  • Dans la colonne Order type, cliquez dans la cellule et sélectionnez asc pour indiquer que les données seront triées par ordre ascendant.

  • Cliquez sur l’onglet Advanced settings pour paramétrer les propriétés avancées du composant et cochez la case Assign FileScale Path et dans le champ FileScale Path, indiquez le chemin d’accès au fichier exécutable correspondant à votre système d’exploitation.

  • Enregistrez le Job et appuyez sur F6 pour l’exécuter.

Une barre de progression apparaît sous le composant tFSSort dans le Job designer pour montrer le pourcentage de progression de l’opération de tri. Cette barre de progression vous montrera la rapidité de l’exécution de l’opération de tri sur un très grand volume de données.

Lorsque le pourcentage de progression atteint les 100%, les données triées en fonction des colonnes city et surname sont écrites dans le fichier de sortie spécifié.
