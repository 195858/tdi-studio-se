Scénario 1 : Réinitialiser l'auto-incrémentation d'une base de données

Le job suivant, à un seul composant, permet de réinitialiser l'auto-incrémentation de la base de données (valeur initiale = 1). Il ne permet pas de générer un flux en sortie car il est généralement utilisé avant l'exécution d'un script.

    • Glissez-déposez le composant tDBSQLRow sur l'espace graphique de modélisation.
    • Dans la vue Basic settings, renseignez les propriétés de la connexion de base de données.
    • Dans le champ Property Type, sélectionnez Repository pour récupérer les informations connexion stockées dans le Repository.
    • Dans le champ Database Driver, sélectionnez Generic ODBC.
    • Dans le champ Database, saisissez le nom de la base de données.
    • Dans les champ Username et Password, saisissez votre identifiant et votre mot de passe de connexion à la base de données.
    • Dans le champ Schema type, sélectionnez Built-in et décrivez la structure de la base de données Talend. Le schéma n'est pas important dans notre cas particulier puisque nous souhaitons réinitialiser l'auto-incrémentation de la table et que nous ne touchons pas aux données.
    • Dans le champ Query type, laissez l'option Built-in. Cliquez sur le bouton [...] à coté du champ Edit schema pour ouvrir le SQLbuilder. Sinon, saisissez le code suivant :
      Alter table <TableName> auto_increment = 1
      directement dans le champ Query.
    • Enregistrez le Job et appuyez sur F6 pour l'exécuter.

L'auto-incrémentation de la base de données est réinitialisée.
