Scénario : Lancer à distance un rapport d'une application Web Talend Integration Suite Administration Center donnée

Ce scénario illustre les opérations collaboratives effectuées pour exécuter à distance un Job de génération de rapport depuis une application Web Talend Integration Suite Administration Center. Pour ce faire, vous devez commiter toutes les métadonnées requises dans le référentiel SVN utilisé par l'application Web Talend Integration Suite Administration Center et installer un serveur d'exécution dans votre système.

Prérequis

Avant de commencer ce scénario, procédez comme suit afin de définir les prérequis :

      • Créez un projet dans votre application Web Talend Integration Suite Administration Center. Les opérations collaboratives pour lancer un rapport seront effectuées dans ce projet. Dans ce scénario, le projet est appelé test_ychen.
        Pour plus d'informations concernant la création d'un projet dans l'application Web Talend Integration Suite Administration Center, consultez la section Ajouter un projet dans le Guide utilisateur de Talend Integration Suite Administration Center.



        • Si vous n'avez pas les droits requis pour créer ou gérer un projet, contactez l'administrateur de votre application Web Talend Integration Suite Administration Center.

      • Etablissez une connexion à partir de votre Studio au référentiel distant stockant le projet susmentionné.
        Pour plus d'informations concernant la création d'une connexion de ce type, consultez la section Se connecter à un référentiel distant dans le Guide utilisateur de Talend Integration Suite.
      • Lancez votre Studio avec cette nouvelle connexion et le nouveau projet créé. pour plus d'informations concernant un tel lancement du Studio, consultez la section Se connecter à un référentiel distant dans le Guide utilisateur de Talend Integration Suite.
      • Dans la perspective Data Profiler de votre Studio, créez un rapport qui sera lancé et généré à distance dans ce scénario. Dans ce scénario, ce rapport est US_State.

Pour plus d'informations concernant la création d'un rapport dans la perspective Data Profiler et la configuration des paramètres relatifs à la génération, consultez la section Créer un nouveau rapport dans le Guide utilisateur de Talend Data Quality.

      • Dans l'éditeur de rapport de la perspective Data Profiler, ajoutez l'analyse, US_State dans ce scénario, sur laquelle vous souhiatez baser votre rapport et saisissez les paramètres afin de générer ce rapport au format PDF et de le stocker dans le répertoire suivant :
        C:\Documents and Settings\ychen\Bureau\TEST_YCHEN\Report_Center.

Pour plus d'informations concernant la configuration des paramètres relatifs à la génération de rapports, consultez la section Créer un nouveau rapport dans le Guide utilisateur de Talend Data Quality.

Si vous n'avez pas créé de base de données de rapports pour garder l'historique des rapports, votre Studio vous demande d'en créer une. Pour plus d'informations, consultez Gérer la base de données de rapports, dans le Guide utilisateur de Talend Data Quality.

Ainsi, les métadonnées requises, comme les analyses, les rapports et les connexions aux bases de données ont été commitées dans le référentiel SVN utilisé par votre application Web Talend Integration Suite Administration Center.

      • Dans le système où est votre Studio, installez et lancez un JobServer. Pour plus d'informations, consultez le chapitre Installation des services dans le Guide d'installation de votre Studio.

Lancer un rapport à distance

Après avoir terminé les prérequis, procédez comme suit pour reproduire ce scénario :

      • Dans la vue DQ Repository de la perspective Data Profiler, cliquez-droit sur le rapport US_State sous le noeud Report et sélectionnez Generate Jobs > Launch a report dans le menu contextuel. Cela crée un Job de génération de rapport.



        • Il est recommandé de vérifier les propriétés (Basic settings) du composant tLaunchDQReports utilisé dans ce Job afin de s'assurer que les chemins d'accès générés pointent bien vers le fichier exécutable de votre Talend Data Quality, vers le rapport qui vous intéresse et vers le projet auquel appartient ce rapport.

      • Dans l'arborescence de votre Application Web Talend Integration Suite Administration Center, développez Job Conductor et cliquez sur Servers pour afficher la liste des serveurs distants d'exécution disponibles pour déployer et exécuter les Jobs.
      • Dans la barre d'outils de la page Servers, cliquez sur Add pour vider les champs du panneau de configuration Execution server.
      • Dans ce panneau de configuration, saisissez et sauvegardez les informations de configuration du JobServer que vous avez installé et lancé dans votre système.
        Pour plus d'informations concernant la configuration requise pour vous connecter à un JobServer, consultez Ajouter un serveur d'exécution dans le Guide utilisateur de Talend Integration Suite Administration Center.

Dans ce scénario, le libellé (Label) du JobServer est Report_Server, son hôte (Host) est 10.42.20.246 et les ports relatifs sont 8000, 8001, et 8888.

      • Cliquez sur Save pour valider cette configuration. Le nouveau serveur s'affiche dans la liste des serveurs dans la page Servers et la coche verte indique que la connexion au serveur est établie.
      • Dans la vue Menu de votre application Web Talend Integration Suite Administration Center, développez Job Conductor et cliquez sur Execution task pour afficher la liste des tâches dans la page Job Conductor.
      • Dans la barre d'outils de la page Job Conductor, cliquez sur Add pour vider les champs du panneau de configuration Execution task.
      • Dans ce panneau de configuration, saisissez les paramètres requis pour l'exécution du Job de génération de rapport que vous avez créé.
      • Dans le champ Label du panneau Execution task, saisissez le nom de la tâche qui sera affichée dans la liste des tâches d'exécution.
      • Dans le champ Project, sélectionnez le projet dans lequel a été créé le Job de génération de rapport. Dans ce scénario, le projet est test_ychen.
      • Dans le champ Branch, sélectionnez la branche du projet, trunk dans ce scénario.
      • Dans le champ Job, sélectionnez le Job de génération de rapport à utiliser.
      • Dans les champs Version et Context, sélectionnez les informations correspondant au Job. Dans ce scénario, les valeurs 0.1 et Default sont utilisées.
      • Dans le champ Execution server, sélectionnez le JobServer qui exécutera la tâche. Ce serveur doit être installé dans votre système. Dans ce scénario, le serveur est Report_Server, que vous avez ajouté à la liste des serveurs.
      • Cliquez sur Save pour valider cette configuration. La nouvelle tâche apparaît dans la liste des tâches dans la page Job Conductor.
      • Dans la liste des tâches, sélectionnez la nouvelle tâche.
      • Dans la barre d'outils du Job Conductor, cliquez sur Generate.
      • Une fois le statut de la tâche sur Ready to send dans la liste des tâches, sélectionnez à nouveau cette tâche et cliquez sur Deploy dans la barre d'outils de la page Job Conductor.
      • Une fois le statut de la tâche sur Ready to run dans la liste des tâches, sélectionnez à nouveau cette tâche et cliquez sur Run dans la barre d'outils de la page Job Conductor.

Une fois l'exécution du Job effectuée, vérifiez dans le dossier de sortie défini dans la vue Generated Report Settings de l'éditeur de rapport dans la perspective Data Profiler de votre Studio.

Dans ce scénario, le chemin d'accès au dossier de sortie est :
C:\Documents and Settings\ychen\Bureau\TEST_YCHEN\Report_Center.
