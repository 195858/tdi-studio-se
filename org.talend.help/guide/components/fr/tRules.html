tRules

Avertissement

Ce composant est disponible dans la Palette de votre studio si vous avez souscrit à l'édition correspondante de Talend Big Data Studio.

Propriétés du tRules

Famille de composant

Processing

 

Fonction

Le composant tRules permet d’appliquer une ou plusieurs règles de gestion métier à un flux de données afin de n’obtenir que les données pertinentes en sortie.

Objectif

tRules permet d’utiliser des règles métier définies dans un fichier Drools au format .xls ou .drl, afin de filtrer des données.

Basic settings

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

DRL/XLS FILE

Sélectionnez le fichier Drools ou Excel contenant les règles que vous souhaitez utiliser dans le Job.

 

Outputs

Configurez les règles à transmettre en sortie.

Schema : Créez ou sélectionnez le schéma à transmettre en sortie.

Rule : Créez ou sélectionnez la règle à utiliser pour la sortie correspondante.

Note

Le champ Import dans le code d’un fichier de règle doit correspondre au nom du projet.

Pour plus d’informations concernant la création de règles, consultez la documentation Drools sur le site suivant : http://www.jboss.org/drools/documentation.html

Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les données de log, aussi bien au niveau du Job qu’au niveau de chaque composant.

Utilisation

Ce composant est un composant intermédiaire dans un flux de données, il nécessite un composant d’entrée et un composant de sortie.

Limitation

n/a

Scénario : Extraire des informations client en fonction de règles métier

Avertissement

Vous devez posséder un fichier de règles métier Drools au format .xls ou .drl pour la réalisation de ce scénario.

Dans ce scénario, les règles Drools sont définies dans un fichier Excel :

Le fichier Excel définit les règles qui seront ensuite appliquées aux données client :

  • la règle RuleAGE récupère les informations des clients ayant entre 30 et 39 ans, et les transmet dans un premier flux de sortie.

  • la règle RuleREGION récupère les informations des clients de la région EMEA et les transmet dans un deuxième flux de sortie.

Avertissement

Lorsque vous utilisez un fichier de règles Drools au format Excel, assurez-vous que toutes les occurrences des noms du projet et du Job du fichier Excel correspondent au nom du projet et du Job dans lesquels sont utilisés les règles.

Ce scénario est un Job de quatre composants ayant pour objectif de lire un fichier contenant des informations client et d’en extraire certains clients en fonction de règles métier préalablement définies dans le fichier de règles Drools.

  • Cliquez et déposez les composants suivants dans l’espace de modélisation : un composant tFileInputDelimited, un composant tRules, et deux composants tLogRow.

  • Dans l’espace de modélisation, double-cliquez sur le composant tFileInputDelimited pour afficher la vue Component et paramétrer ses propriétés.

  • Dans la liste Property type, sélectionnez le mode Built-in et renseignez les champs suivants manuellement.

  • Cliquez sur le bouton [...] à coté du champ File Name pour indiquer le chemin d’accès au fichier d’entrée. Le fichier source utilisé dans cet exemple s’appelle client et contient des informations sur les clients.

  • Configurez les champs Row et Field separator, Header, Footer et Limit, si nécessaire. Pour plus d’informations concernant le composant tFileInputDelimited, consultez la section intitulée « tMDMOutput »

  • Dans la liste Schema, sélectionnez Built-in et cliquez sur le bouton Edit schema pour définir le schéma du fichier comme suit :

Dans cet exemple, notre fichier source est composé de quatre colonnes : id, name, age et region.

  • Cliquez sur OK pour valider vos changements et fermer la boîte de dialogue.

  • Dans l’espace de modélisation, double-cliquez sur le composant tRules pour afficher la vue Component et paramétrer ses propriétés.

  • Dans la liste Property type, sélectionnez Repository si vous avez créé une connexion à votre fichier de règles dans les métadonnées du Repository. Si vous n’avez pas créé de connexion au fichier de règles dans le Repository, sélectionnez Built-in dans la liste Property type et renseignez les champs suivants manuellement.

    Pour plus d’informations concernant la création d’une connexion au fichier de règles, consultez le Guide utilisateur de Talend Data Integration Studio.

  • Dans le tableau Outputs, ajoutez deux lignes de sortie pour créer deux flux de sortie différents utilisant deux règles de gestion métier différentes prédéfinies dans le fichier de règles.

  • Cliquez dans le champ Schema de la première ligne et cliquez sur le bouton [...] qui apparaît.

  • Dans la boîte de dialogue qui apparaît, renseignez le nom du schéma à transmettre au flux de sortie. Dans ce scénario, appelez ce schéma rule_region. Cliquez sur OK.

  • Dans la deuxième boîte de dialogue, paramétrez le schéma de sortie. Dans ce scénario, récupérez le schéma d’entrée. Cliquez sur OK.

  • Cliquez dans le champ Rule de la première ligne et cliquez sur le bouton [...]. La boîte de dialogue [Rule] s’ouvre.

  • Dans la boîte de dialogue, choisissez l’action à effectuer :

    - View Rules pour ouvrir le fichier de règles en lecture seule, ou

    - Select a rule from repository pour sélectionner une règle du fichier stocké dans le Repository.

  • Dans la liste Rule, sélectionnez la règle que vous souhaitez appliquer au premier schéma de sortie, RuleREGION, et cliquez sur OK.

    La règle sélectionnée apparaît dans la colonne Rules.

  • Configurez la deuxième ligne de la même manière, en nommant le schéma rule_age et en sélectionnant la règle RuleAGE.

  • Dans l’espace de modélisation, double-cliquez sur chaque composant tLogRow pour paramétrer leurs propriétés dans la vue Component. Pour plus d’informations, consultez la section intitulée « tLogRow »

  • Enregistrez votre Job et appuyez sur F6 pour l’exécuter.

La console Run affiche les deux flux de sortie : celui contenant les informations sur les clients ayant entre 30 et 39 ans, et celui contenant les clients de la région EMEA.

Exécuter un Job avec un tRules utilisant des règles métier définies dans un fichier Excel permet d'afficher les résultats d'exécution dans la console, avec un message d'avertissement. Pour supprimer le message d'avertissement, procédez comme suit :

  1. Ouvrez le fichier Excel dans Microsoft Excel ou OpenOffice.

  2. Supprimez toutes les entrées de la liste des noms :

    Dans Microsoft Excel :

    • Dans la barre de menu, sélectionnez Formulas puis Defined Names et cliquez sur Name Manager.

    • Sélectionnez toutes les entrées de nom dans la liste et cliquez sur Delete.

    Dans OpenOffice :

    • Dans la barre de menu, sélectionnez Insert > Names > Define.

    • Sélectionnez toutes les entrées de nom dans la liste et cliquez sur Delete.

Si vous exécutez le Job après cette édition du fichier Excel, les résultats seront affichés dans la console, sans message d'avertissement.
