Note

Propriétés du tGoogleAddressRow

Famille de composant

Data Quality/Address

 

Fonction

Le composant tGoogleAddressRow accède à l'API Google Geocoding via une requête HTTP afin d'obtenir des coordonnées géographiques ainsi que d'autres informations géographiques, selon les informations relatives à l'adresse que vous avez fournies.

Note

Pour plus d'informations concernant l'API Google Geocoding, consultez The Google Geocoding API (en anglais).

Objectif

Le tGoogleAddressRow vous permet de convertir des adresses d'un format lisible par l'homme en coordonnées géographiques et autres informations géographiques.

Basic settings

Pause(in 1/10 seconds)

Temps de pause entre les requêtes HTTP pour les enregistrements relatifs aux informations géographiques. Par défaut, la valeur de ce champ est 1.

 

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

 

 Address column

Dans cette liste, sélectionnez un paramètre à utiliser lors de l'envoi de la requête Geocoding vers l'API Google Geocoding. Les paramètres de la liste sont prédéfinis dans le schéma des données d'entrée.

  Die on error Cochez cette case pour arrêter l'exécution du Job en cas d'erreur. Décochez-la afin d'ignorer les lignes en erreur et terminer processus sans ligne en erreur.

 Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Dans le champ Number of parallel executions, vous pouvez :

- Appuyer sur Ctrl+Espace et sélectionner la variable de contexte dans la liste.

Pour plus d’informations, consultez le Guide utilisateur de Talend Data Integration Studio.

Avertissement

Les actions sur la table (Action on table) ne sont pas disponibles avec la fonction de parallélisation. Ainsi, pour créer une table, vous devez utiliser un composant tCreateTable.

Avertissement

Lorsque l’exécution parallèle est activée, il n’est pas possible d’utiliser les variables globales pour retrouver les valeurs de retour dans un sous-job.

Utilisation

Ce composant est un composant intermédiaire. Il nécessite un composant d'entrée, un composant de sortie et un lien Main Row.

Scénario : Obtention d'informations géographiques détaillées à l'aide d'adresses

Ce scénario décrit un Job a trois composants envoyant une requête HTTP à l'API Google Geocoding et obtenant des informations géographiques détaillées, à partir d'une adresse fournie.

Déposer et relier les composants

  1. Déposez les composants suivants de la Palette dans l'espace de modélisation graphique : tFixedFlowInput, tGoogleAddressRow et tLogRow.

  2. Reliez le tFixedFlowInput au tGoogleAddressRow et le tGoogleAddressRow au tLogRow à l'aide de liens Main.

Configurer le composant d'entrée

  1. Double-cliquez sur le composant tFixedFlowInput afin d'afficher sa vue Basic settings et définir ses propriétés.

  2. Cliquez sur Edit schema pour ouvrir une boîte de dialogue affichant la structure des données d'entrée et définir les données que vous souhaitez passer au composant tGoogleAddressRow. Dans ce scénario, passez deux colonnes, AddressID et Address.

  3. Cliquez sur OK pour fermer la boîte de dialogue.

  4. Cliquez sur Use Inline Table dans la zone Mode de la vue Basic settings du tFixedFlowInput.

  5. Dans la zone Mode de la vue Basic settings du composant tFixedFlowInput, cliquez sur le bouton [+] afin d'ajouter une colonne à la table.

  6. Renseignez la colonne AddressID en saisissant 1 et la colonne Address en saisissant "500 Oracle Parkway,Redwood Shores, CA 94065".

  1. Double-cliquez sur le composant tGoogleAddressRow pour afficher sa vue Basic settings et définir ses propriétés.

  2. Cliquez sur Sync columns pour récupérer le schéma défini dans le tFixedFlowInput.

  3. Sélectionnez Address dans la liste Address Column et laissez les autres paramètres tels qu'ils sont.

  4. Double-cliquez sur le tLogRow pour afficher sa vue Basic settings et définir ses propriétés.

  5. Cliquez sur Sync columns pour récupérer le schéma défini dans le tFixedFlowInput.

  6. Sélectionnez Vertical (each row is a key/value list) dans la zone Mode et Print unique name dans la zone Title printing mode.

Sauvegarder et exécuter le Job

  • Sauvegardez votre Job et appuyez sur F6 pour l'exécuter.

Les informations géographiques de l'adresse 500 Oracle Parkway,Redwood Shores, CA 94065 sont obtenues de la base de données Google Map et affichées dans la console.
