Scenario: Reading data in an MDM database

This scenario describes a two-component Job that reads data in an XML file in an MDM database. The data is fetched and displayed in the log console.

    • From the Palette, drop tMDMInput and tLogRow onto the design workspace.
    • Connect the components together using Row Main links.
    • Double-click tMDMInput to open the Basic settings view and define the component properties.
    • In the Property Type list, select Built-in to complete the fields manually. If you have stored the MDM connection information in the repository metadata, select Repository from the list and the fields will be completed automatically.
    • In the Schema Type list, select Built-in and click the three-dot button next to Edit schema to open a dialog box in which you can define the structure of the data you want to read in the MDM database.
    • The data is collected in a three column schema of the type String: ISO2Code, Name and Currency. Click OK.
    • In the URL field, enter the MDM database URL, between inverted commas.
    • In the Username and Password fields, enter your login and password to connect to the MDM database.
    • In the Version field, enter the name of the perspective you want to access, between inverted commas. Leave this field empty to display the default perspective.
    • In the Entity field, enter the name of the entity that holds the data you want to read, between inverted commas.
    • In the Data Container field, enter the name of the directory that holds the data you want to read, between inverted commas.
    • In the Component view, click on Advanced settings to set the advanced parameters.
    • In the Loop XPath query field, enter the structure and the name of the XML node on which the loop is to be carried out, between inverted commas.
    • In the Mapping area and in the XPath query column, enter the name of the XML tag in which you want to collect the data, between inverted commas, next to the corresponding output column name.
    • In the design workspace, click on the tLogRow component to display the Basic settings in the Component view and set the properties.
    • Click on Edit Schema and ensure that the schema has been collected from the previous component. If not, click Sync Columns to fetch the schema from the component.
    • Save the Job and press F6 to run it.

The list of different countries along with their codes and currencies is displayed on the console of the Run view.
