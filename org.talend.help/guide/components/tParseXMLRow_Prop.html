tParseXMLRow

tParseXMLRow Properties

Component family
XML
Function
tParseXMLRow reads an input XML field and extracts desired data.
Purpose
tParseXMLRow opens an input XML field, reads structured data directly without having first to write it out to temporary files, and sends data as defined in the Schema to the next component, via a Row link.
Basic settings
Property type
Select either Built-in or Repository.
Built-in: No property data stored centrally.
Repository: Properties are stored in a Repository file. When selected, the following fields are pre-defined using fetched data.
Schema type and Edit Schema
A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository.
 
Built-in: You create and store the schema locally for this component only. Related topic: Setting a built-in schema of Talend Open Studio User Guide
 
Repository: You have already created the schema and stored it in the Repository. You can reuse it in various projets and job flowcharts. Related topic: Setting a repository schema of Talend Open Studio User Guide
XML field
Name of the XML field to be processed.
Related topic:Defining variables from the Component view of Talend Open Studio User Guide
Loop XPath query
Node of the tree, which the loop is based on
Mapping column/XPath Query
Column reflects the schema as defined by the Schema type field
XPath Query: Enter the fields to be extracted from the structured input.
 
Limit
Maximum number of rows to be processed. If Limit = 0, no row is read or processed.
Usage
You must use this component as an intermediate component.
Limitation
For the time being, can not read huge XML input data.
